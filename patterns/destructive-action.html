Destructive action

Two-step delete flow — trigger opens a confirmation modal, never deletes immediately. The delete button is ghost or secondary as a trigger; only the danger variant appears inside the confirmation footer.

Examples

Trigger — ghost button opens confirmation modal
Delete document
Confirmation modal — Cancel (ghost) left, Delete (danger) right
Preview confirmation modal

This document will be permanently deleted. Related insights will be updated.

Cancel Delete

Rules

  • Never delete immediately on click — always a confirmation modal.
  • deletable prop must be true before showing the trigger. Never show a delete button the user cannot use.
  • Delete button is ghost or secondary as a trigger; only the danger variant appears inside the confirmation footer.
  • On confirm, navigate to the parent page — never leave the user on a now-empty page.
  • Body copy acknowledges downstream effects: "Related insights will be updated."

Copy

CTA
"Delete" — never "Remove", "Erase", or "Yes, delete"
Cancel
"Cancel" — never "No", "Go back", or "Keep"
Body
One sentence. "This document will be permanently deleted. Related insights will be updated."
