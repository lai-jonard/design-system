Person entity modal

Detail, add, and confirm modal states for a named person — used in Founders (Company Manual) and People (Employment). Clicking a person row opens a detail modal; the remove trigger within that modal opens a confirmation modal.

Accessibility

  • Set heading="" to suppress the visible title bar — the person's name in the body serves as the visual heading.
  • Set aria-label on the modal element to the person's name.
  • Screen readers announce the dialog name on open, then read body content sequentially.
modal.setAttribute('aria-label', person.name);

Rules

  • Ghost + --la-color-danger-text for the remove trigger in the detail modal — not filled danger.
  • Filled danger variant is reserved for the confirmation step only.
  • Subtitle in the add modal is always visible — never hidden, even before a name is entered.
  • Use display: contents on the footer wrapper div so margin-right: auto works correctly to push the remove button to the left.
  • Avatar initials update live as the user types in the add modal.
click la-person-item row → populate + open detail modal click "Remove" in detail modal → close detail modal → open confirmation modal click "Remove" in confirmation modal → remove from data array → re-render la-person-list
