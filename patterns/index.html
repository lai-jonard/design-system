Patterns

Standardised interaction patterns used across LawAdvisor AI. Every pattern here has shipped in at least one sprint prototype and reflects how we handle the interaction — not how we'd like to handle it hypothetically.

Interaction
Destructive actionv1 ready

Two-step delete flow — trigger opens a confirmation modal, never deletes immediately.
Upload / Replacev1 ready

Adding a new document or replacing an existing one. Upload modal, processing state, success and failure transitions.
Confirmation modalv1 ready

Any hard-to-reverse action. Single heading, one-sentence body, two-button footer with consistent button order.
Insight row → modalv1 ready

Clicking an insight row fires la-insight-select. Consumer populates and opens la-insight-modal.
Empty statesv1 ready

Inviting, actionable empty states. Placement, copy tone, and when to show inline vs. full-panel.
Flag bannersv1 ready

Persistent in-page alerts for flagged or processing document states. Placement, variants, and dismissal rules.
Processing statesv1 ready

How the UI behaves while a document or action is processing. Spinner placement, copy, and state transitions.
Inline metadatav1 ready

Middle dot (·) as the separator for horizontal label runs. Implementation rules and conditional visibility.
People
Person entity modalv1 ready

Detail, add, and confirm modal states for a named person. Header anatomy, footer conventions, accessibility.
