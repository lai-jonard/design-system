Processing states

UI behaviour while an upload, analysis, or background action is in progress. Show la-spinner inline — never block the entire page. Existing content stays visible throughout.

Examples

Inline spinner with label
Analysing document…
Processing widget
Failure state — warning alert with retry
Analysis failed. Try uploading the document again. Try again

Rules

  • Show la-spinner inline — never block the entire page with a full-screen overlay.
  • Existing content stays visible during processing. Going dark or hidden would alarm users.
  • Copy is present-tense: "Analysing document…", "Uploading…"
  • On failure: show la-alert variant="warning" with a retry CTA — never silently revert.
