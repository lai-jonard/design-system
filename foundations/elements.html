Foundations · Elements

Elements

The system styles a small set of native HTML elements directly so that plain markup looks right by default — no Lit component needed. These rules live in la-base-root.css and apply globally wherever the file is loaded.

If an element needs interactivity, composition, or variants exposed as attributes, build a Lit primitive instead. Elements documented here are pure visual defaults — semantic markup wearing the system's voice.

Sections are ordered by how often each element appears in real product surfaces, from Essential (every page has them) down to Niche (specialised contexts only). Each section opens with a short eyebrow tag indicating its priority and typical use.

Essential · every page has links

Anchors <a>

Native links, styled by default. Three resting tones — the bare element, the .link-muted modifier for tertiary use, and .link-inline for links sitting mid-sentence.

Always underlined. Text colour stays muted at rest; only the line colour shifts on hover. The line is the affordance — colour is just the tone.

Default — <a href="…">

Three resting tones — at a glance
TreatmentText colourLine colour (rest)Line colour (hover)Use
<a> --la-color-text-secondary --la-color-border --la-color-primary Standalone CTAs and navigation — "View details", "Read the full doc"
.link-muted --la-color-text-muted --la-color-border --la-color-text Tertiary links inside dense rows or quiet navigation

Behavioural extras (external indicator, button fallback, disabled state) are not in scope here. Native <a> covers most uses. For external links, add target="_blank" rel="noopener noreferrer" by hand.

Common · technical docs, tooltips, references

Inline code <code>

Monospace, subtle muted background, tight horizontal padding. Use for inline references to props, files, tokens, and short snippets.

Short — single sentence with a few references
Reference --la-color-primary in your component, or import the bundle from dev/bundle.js. The render() method must include ${iconStyles} if you use Phosphor icons.
Elaborate — code references woven through walkthrough copy

A new Lit component starts with two imports — LitElement from lit and customElement from lit/decorators.js. Decorate the class with @customElement('la-name'), declare reactive properties via @property(), and pull tokens from src/styles/tokens.js into the static styles array so design tokens reach the shadow root.

Reference tokens through CSS custom properties — var(--la-color-primary) for primary action, var(--la-space-md) for spacing, var(--la-radius-md) for corners. Never hardcode hex codes or pixel values: if a token isn't on the scale, that's a design conversation in tokens.ts, not a one-off override.

Phosphor icons need two pieces inside shadow DOM. Include ${iconStyles} at the top of render() so the @font-face rules reach the shadow tree, then use <i class="ph ph-file"> for each glyph. Outline weight (ph) for the resting state, fill (ph-fill) for active or selected — never both at once.

Niche · command palettes, keyboard-shortcut hints

Keyboard chord <kbd>

Soft chip in muted bg with hairline border. Inherits the body font (Inter) so the command glyph (⌘) aligns cleanly with adjacent letters; tabular numerals keep digits equal-width. Travels with text and surfaces alongside actions — buttons, menus, search inputs.

In a sentence
Press ⌘ + K to open the command palette, Esc to close, or Enter to select.
