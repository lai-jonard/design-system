Native form controls

What raw <input>, <textarea>, <select>, <button>, <label>, <input type="radio">, and <input type="checkbox"> look like in pages that load la-base-root.css on top of la-tokens-root.css. Most of these are unstyled today — the base file currently covers content elements (anchors, headings, lists, code, etc.) but not form controls. Reach for an la-* primitive whenever one exists; the gaps are flagged in the right column of each section so engineering knows what they're getting.

<input>

The browser's native input. Not styled by la-base-root.css — the system delegates this to la-input, which carries label, hint, error, icon-slot, and consistent chrome. Reach for the primitive.

Native (browser default)
la-input (recommended)
When you must use native. Some LiveView form serialisers expect raw <input> + name attributes. If you're stuck with native, document it in the surrounding component and queue the swap to la-input when the serialiser supports it.

<textarea>

Long-form free text. No system primitive yet — Sprint 4's "short answer" steps stay on a single-line la-input; longer text inputs use native. The gap is logged.

Native textarea (current path)
Gap to schedule. The system needs la-textarea with auto-grow, character counter, and the same label/hint/error rhythm as la-input. Not in Sprints 1–4 scope; queued for build when a long-form text step lands.

<select>

Single-choice dropdown. la-select + la-option is the system primitive (currently 🟡, awaiting /document + /ready). The native control is acceptable inside an embedded LiveView form when the serialiser hasn't been migrated; otherwise use the primitive.

Native select
la-select (recommended) Delaware California New York

<button>

Native button is rarely the right choice. la-button carries five variants, three sizes, focus rings, hover states, motion, and full keyboard semantics. Use the primitive.

Native button
la-button (recommended) System primitive
Exception. Inline icon-only triggers inside dense docs chrome (the docs nav toggle, sidebar collapse) use <button> with locally-scoped styling. Don't replicate this for product surfaces.

<label>

Always pair a label with its control. The system uses native <label> with for bound to the input id — that gives screen readers and click-to-focus behaviour for free. Style is owned by the form context, not by la-base-root.css.

See Forms — field anatomy for the canonical label rhythm (size, weight, colour, required marker).

<input type="radio">

Single-choice from a small set. No system primitive yet — la-radio-group is logged as a known gap that the unlock flow's yes/no step needs (currently rendered as click-handler-on-spans, an anti-pattern flagged in la-unlock-panel).

Native radio group (current path)
Gap to schedule before Sprint 4 ships properly. The unlock-flow yes/no step currently renders clickable <span>s with no radio semantics. la-radio + la-radio-group is logged in handoffs/readiness.md. Sprint 8 (contract-first add-person modal) and future IP-filing steps both compound the demand.

<input type="checkbox">

la-checkbox is the system primitive. Native <input type="checkbox"> is acceptable only inside la-data-table selection columns where the table primitive owns the chrome.

Native checkbox
la-checkbox (recommended)

<fieldset> / <legend>

Group related fields. Native semantics — screen readers read the legend before each grouped field. Restyle locally with border: none + padding: 0; the legend can carry the same chrome as a form label.
