Foundations · Typography

Typography

Inter across the system. Hierarchy comes from size, weight, and grey scale — never colour. The scale has nine steps; pick the smallest one that reads comfortably for the role. If the value you need isn't on the scale, that's a design conversation, not a new size.

Principles

From CLAUDE.md's design-language rules and the way the system actually reads in product. Four non-negotiables.

Inter only System-wide, loaded globally. --la-font-family. Monospace (ui-monospace) is allowed for code, token names, and IDs — never for body or headings.
Headings are weight 600 — never 700 Bold (700) is in the token scale for edge cases and direct imports, but every heading in every component uses semibold. Bolder heading weight crowds adjacent body copy.
Hierarchy through weight + grey scale Never through colour. Blue/red/green mean something specific; using them for emphasis dilutes their signal.
Two-register capitalisation, never all-caps Structural labels (navigation, buttons, table headers) use Title Case. Explanatory copy (insight headlines, helper text, empty states) uses sentence case. See the Copy & Capitalisation page.

The scale

Nine steps from 10px to 24px. The scale favours tight increments at small sizes (10/11/12/13/14) so you can pick the right density for dense chrome, then jumps to 16/18/20/24 for hierarchy. Each token name maps to one value — never override per-instance.

--la-font-size-md 12px Dense chrome — table cells, inline controls, tight nav items.
--la-font-size-sm 11px Meta text — captions, timestamps, secondary labels.
--la-font-size-xs 10px Smallest legible step — status chips, badges.

Weights

Four weights. In product, only three are in everyday use: normal for body, medium for subtle emphasis and nav/button labels, semibold for headings. Bold (700) is available for edge cases outside component UI (logo lockups, hero moments) but never for standard headings.

Line height

One value, applied globally on :host: --la-line-height: 1.5. Inherit it — don't override unless you're dealing with a specific layout need (dense table rows, icons centred against text).

Tight body copy (<1.4) looks crowded. Roomy copy (>1.6) breaks up headings. 1.5 sits in the common-sense middle for Inter at every size on the scale.

Headings

Four levels, each with a clear job. Don't skip levels for visual effect — semantic hierarchy drives accessibility and document outlines.

LevelSize tokenWeightRole
H1 --la-font-size-3xl (20px) or -4xl (24px) 600 Page title. One per page. Rarely used inside components.
H2 --la-font-size-2xl (18px) 600 Major section heading — modal title, dialog heading, settings group.
H3 --la-font-size-xl (16px) 500 or 600 Sub-section inside a component — alert content, section of a widget.
H4 --la-font-size-lg (14px) 600 Widget titles, card headlines — the small-but-structural heading.
One page title rule. Every page has exactly one H1. In this system that's either the la-app-shell page header or the route's page component — not anything inside a widget. If you're tempted to add a second H1 to emphasise something, use an H2 and raise the weight.

Body copy

Rules from CLAUDE.md — the ones that matter most for type.

  • One to two sentences max. If you need three, the surrounding UI is missing an affordance or a link out.
  • Active voice. "We added a clause" not "A clause was added."
  • British English. Organisation, authorised, licence (noun), colour.
  • Verb-first CTAs, sentence case. "Send for signature", "Add employee", "View details".
  • Use the person's name. Never "the recipient" or "the employee" when a name is known.

Body default: --la-font-size-base (13px), font-weight: 400, colour --la-color-text. For secondary body (helper text below an input, meta lines under a heading), drop to --la-color-text-secondary — don't shrink the size unless the role actually changes.

Labels & meta

Two sizes do most of the small-text work: --la-font-size-sm (11px) for meta and captions, --la-font-size-xs (10px) for chips and badges. Don't reach lower — 9px and below fail a11y contrast on typical product screens.

Weight for labels: 600 when they need to pop off a surface (chips, eyebrows), 400 when they're descriptive (captions, timestamps). Colour drives emphasis: --la-color-text-muted for most meta, --la-color-text-secondary when the meta carries real meaning.

In code

Every Lit component imports tokens from src/styles/tokens.js and includes it at the top of static override styles. That puts the custom properties in scope inside the shadow root.

import {tokens} from '../styles/tokens.js'; static override styles = [ tokens, css` .headline { font-size: var(--la-font-size-lg); font-weight: var(--la-font-weight-semibold); color: var(--la-color-text); } .body { font-size: var(--la-font-size-base); color: var(--la-color-text-secondary); } .meta { font-size: var(--la-font-size-sm); color: var(--la-color-text-muted); } `, ];

Don'ts

Don't use weight 700 for headings. Semibold (600) is the heading weight system-wide. 700 only belongs in external/marketing surfaces or logo lockups — never in component UI.
Don't use all-caps. Not for labels, not for eyebrows, not for section headings. Use sentence case and let size + weight + grey scale carry the hierarchy.
Don't use colour for hierarchy. Blue, red, green, amber all have semantic roles (processing, danger, success, warning). Using them to draw attention to a heading dilutes the real signal.
Don't hardcode font-size: 13px. Use the token. If the value you need isn't on the scale, it's a design conversation — not a new pixel.
Don't introduce new font sizes between the steps. The scale is deliberately coarse above 14px. If you want something "slightly bigger than 14", pick 16 — or challenge the design.
