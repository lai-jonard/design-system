Foundations

The rules that govern every component. Read this once and the rest of the system makes sense. Every value on this page references a real token — the source of truth is src/styles/tokens.ts.

How we work

The system runs on two core workflows, both codified as slash commands. /build takes a product request through scout → align → build with hard stops for user approval. /audit runs eight gates against an existing component and flags what's missing to reach v1.

Every component lives as three artefacts: Lit source in src/, a docs page under docs-src/components/ (with in-progress drafts staged at dev/docs-drafts/), and a problem statement in design-language/components.md. Readiness status is tracked in handoffs/readiness.md. Cross-component decisions get numbered files in decisions/.

Tone & voice

Calm authority. Steady and confident, never urgent. British English. Active voice. Verb-first CTAs in Title Case — "Send for Signature", "Add Employee", "View Details" — not noun phrases, never ALL CAPS. Explanatory copy (insight headlines, helper text, empty states) uses sentence case. See the Copy & Capitalisation page for the full rule.

No jargon: "start" not "commence", "check" not "verify", "send" not "submit". Empty states invite action. Success moments are brief ("Done.", "All sorted."). Errors are honest and unflustered.

Accessibility

Baseline expectations for every component. These are Gate 7 of /audit — a component can't reach 🟢 without passing them.

  • Semantic elements — <button>, <a>, <input> — over <div> with a click handler.
  • Icon-only buttons must carry an aria-label. la-icon-button requires its label prop.
  • Form controls need a visible label, aria-label, or aria-labelledby.
  • Visible focus style always. Never outline: none without an equivalent replacement.
  • Colour plus something else — icon, copy, position — for meaning. Never colour alone.
  • Motion respects prefers-reduced-motion. The rule lives in motion.css and the shared spinner styles.

Colour

Two layers. Core primitives (the 17 hexes below, organised by hue) are the only place a hex literal lives. Semantic tokens (everything else — --la-color-danger, --la-color-bg-subtle, …) are var() aliases that resolve to a primitive. Always reach for the semantic name in components — the primitive layer exists so the relationship between intent (danger) and hue (red) is explicit and editable in one place.

Other rules: the primary action colour (--la-color-primary) only appears on interactive elements. Coloured surfaces signal state; they don't decorate. Never hardcode hex — always reference a token.

Core palette

Hue scales. Add a primitive only when ≥2 semantic tokens resolve to the same value; single-consumer values stay inline. Naming follows numeric scale (-50 tint, -500 base, -700 darker) so adding -300 or -900 later is non-breaking.

Radius

Corner radii follow the concentric rule — a panel's outer radius equals its padding plus the radius of the children inside it. Nested shapes share a centre; the eye reads one considered system instead of two rounded things.

Shadows

Three elevation steps. Per CLAUDE.md, shadows only appear on floating elements — modals, dropdowns, toasts, ambient trays. Never on static cards sitting inside a container; that's what borders and surface tints are for.

Typography

Inter, loaded globally. Headings are font-weight: 600 — never 700. Hierarchy comes from weight + grey scale, never colour. No all-caps outside narrow system chrome (eyebrow labels, table headers).

--la-font-size-3xl / 20px The quick brown fox
--la-font-size-2xl / 18px The quick brown fox
--la-font-size-xl / 16px The quick brown fox
--la-font-size-lg / 14px The quick brown fox jumps over the lazy dog.
--la-font-size-base / 13px Body copy — 1–2 sentences max, per the copy rules.
--la-font-size-md / 12px Dense chrome — table cells, inline controls.
--la-font-size-sm / 11px Meta, captions, secondary text.
--la-font-size-xs / 10px Smallest legible step — status chips, badges.

Weight tokens: --la-font-weight-normal (400), --la-font-weight-medium (500), --la-font-weight-semibold (600), --la-font-weight-bold (700). Base line-height --la-line-height (1.5).

Motion

Source of truth is motion.css. Utility classes + tokens handle every animation — never handwritten keyframes, never ease-in.

  • GPU-safe only — animate transform and opacity. Never height, width, top, left, background, or color.
  • Never scale from zero — minimum starting scale is 0.96.
  • Max 300ms for repeated interactions, 500ms for anything triggered often.
  • Stagger: 40–60ms between list items.
  • Reduced motion: motion.css automatically collapses all durations to 0ms when prefers-reduced-motion is set.

Icons

Phosphor only. Regular for resting state (ph ph-file), fill for active or emphasised (ph-fill ph-check-circle), bold for weighted typographic use.

Icons always inherit colour from their parent — never hardcode. Set color on the container; the glyph follows. Pair colour with something else (position, copy, weight) for meaning — never colour alone, per the accessibility baseline.

Any component using icons inside shadow DOM must include ${iconStyles} at the top of render() so the Phosphor CSS reaches the shadow root.

Preview pages must also link the Phosphor CSS in <head> — the @font-face registers globally, while iconStyles covers the component's shadow tree.

Responsive

We build for desktop first, degrade cleanly to tablet, adapt to mobile. Component-level layout uses container queries where possible rather than viewport breakpoints — widgets should react to their own width, not the window's.

  • container-type: inline-size on the host, then @container (max-width: 420px) {…} inside.
  • Truncate long text with text-overflow: ellipsis on any row that stacks inside a variable-width container.
  • Use flex-shrink: 0 on fixed-dimension children (icons, avatars, spinners) so they don't deform when siblings grow.
  • Full-page layouts use media queries in the page component itself, not in downstream atoms.
Gotcha: container-type: inline-size creates a containing block for absolutely positioned children. Use position: fixed for overlays that need to escape, or hoist them out of the container.

Troubleshooting

Icons not rendering inside a component

Two-part setup. The page <head> must link the Phosphor CSS (registers @font-face globally). The component's render() must include ${iconStyles} (applies the .ph.ph-* class rules inside the shadow root).

"npm run dev" not loading the bundle

The server root is dev/. Docs drafts at dev/docs-drafts/ link ../bundle.js. Earlier prototype pages may use ../../dev/bundle.js; update to the shorter path when you next touch them.

Hardcoded px or hex sneaking back in

Both /audit Gate 8 and CLAUDE.md prohibit this. If the value you need isn't in tokens.ts, add a token — don't inline the pixel. Intrinsic component dimensions (a 14px spinner, a 32px icon tile) live inside the primitive; everything else references a token.

Motion looks wrong in Firefox or Safari

Probably animating a layout property by accident. GPU-safe only — transform and opacity. Check the Motion section above for the full rule set.
