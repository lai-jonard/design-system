Foundations / Page layouts

Page layouts

Concrete templates for the four surfaces sprints 1–4 build on top of la-app-shell: the dashboard, the legal area page, the unlock host page, and the standalone sign-in. Each template names the grid, the max-widths, the spacing scale, and what owns each region. Numbers come from the sprint handoffs and the design-language layout doc.

App shell (Sprint 1)

Three regions: sidebar (left, fixed) · main (centre, scrollable) · aside (right, optional, drawer at narrow viewports). la-app-shell owns the chrome — header bar, breadcrumb, back button, aside toggle. The page composing it provides slot content.

Legal area page (Sprint 3)

Two-column inside the app shell's main slot — insights on the left, the actions widget on the right. The aside slot of la-app-shell stays separately reserved for sub-tab content (Sprint 7+). Both columns scroll inside main; the shell handles outer chrome.

Unlock host page (Sprint 4)

Full-page blocking flow. The locked legal area page hosts la-unlock-panel as the only main-slot content — sidebar + breadcrumb stay visible, but the area's tabbed insights view is replaced entirely until the unlock completes.

SidebarVisible. Locked areas show lock icon; the area being unlocked shows as active.
Main slotHolds la-unlock-panel only. No sub-tabs, no actions widget, no insights — the unlock flow takes the full content area.
Aside slotEmpty. Unlock is a focused task.
WidthThe panel fills the main slot; an inner max-width keeps line lengths readable. la-unlock-panel handles its own internal layout.
ExitSprint 4 framework only — back-to-dashboard via breadcrumb. Per-area destinations on completion arrive in Sprints 7–10.

Standalone (sign-in, magic-link landing)

Off-shell layout. Sprint 1 sign-in is the canonical example — centred panel, no sidebar, no breadcrumb. Used for any surface where the user isn't signed in or hasn't entered the app proper.

Shared principles

  • White first. Surfaces are white or near-white; the page background is --la-color-bg-page. No coloured section backgrounds.
  • Borders structural, not decorative. --la-color-border-light for layout dividers; --la-color-border for interactive components.
  • Shadows only on floating elements. Modals, dropdowns, toasts — never on static cards.
  • Library-level max-width is 1280px (general content). Sprint 2 dashboard tightens to 1120px specifically; Sprint 3 legal area uses the natural width inside la-app-shell's main slot. Don't widen beyond 1280px without a layout review.
  • Card grids use auto-fill. Generic card layouts: grid-template-columns: repeat(auto-fill, minmax(300px, 1fr)); gap: 14px; — never fixed column counts. The Sprint 2 dashboard is the exception: fixed because the legal areas are a known set.
  • Generous padding. Content is never crowded. Defaults — content area top padding 24px, horizontal 28px; Sprint 2 widens dashboard top padding to 80px.
  • Responsive degrades cleanly. Desktop ≥1024px, tablet 768–1023, mobile <768. Sprints 1–4 ship desktop only; tablet and mobile are deferred reworks. Container queries run at component level (la-app-shell aside drawer flip), media queries at page level.
