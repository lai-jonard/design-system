Timestamps

Guidelines for displaying dates and times across LawAdvisor AI product surfaces.

The core rule

Use relative time when recency is the signal. Use exact time when the specific moment matters. Never show a tooltip on a timestamp that already displays exact information.

Display formats

Relative time

Use when the primary signal is how recent something is, not when exactly it happened.

Age Format Example
< 1 minute Just now "Just now"
1–59 minutes N minutes ago "14 minutes ago"
1–23 hours N hours ago "3 hours ago"
1 day Yesterday "Yesterday"
2–6 days N days ago "4 days ago"
7+ days Exact date See exact formats below

Always pair relative time with a la-tooltip showing the full datetime. Relative time is lossy — the tooltip restores precision on demand without cluttering the layout.

<la-tooltip tip="12 May 2026, 2:14 pm" appearance="light" size="sm">
  <span class="timestamp">2 hours ago</span>
</la-tooltip>

Exact time (same-day)

Use when the specific hour matters and the date is already established by context (e.g. within a conversation thread where a date separator is shown above).

Format: 2:14 pm

No tooltip. The displayed value is already precise.

Exact date + time

Use when the full temporal context matters — older messages, activity logs, audit trails.

Format: 12 May 2026, 2:14 pm

No tooltip. The full information is already visible. Adding a tooltip repeating the same value adds no signal and creates noise.

Date only

Use when time doesn't matter — date separators in threads, event dates, document update dates.

Format: 12 May 2026

Tooltip rule — summary

Display format Tooltip? Why
"2 hours ago" ✓ Yes — show exact datetime Relative text hides precision; tooltip restores it
"Yesterday" ✓ Yes — show exact date + time Same — relative label is ambiguous after midnight
"2:14 pm" ✗ No Time is already shown
"12 May 2026, 2:14 pm" ✗ No Full datetime is already shown — tooltip would repeat it
"12 May 2026" ✗ No (usually) Date is explicit; add only if the time is also relevant

The la-chat-message component exposes timestamp-title for the tooltip value. Only set it when the timestamp prop shows relative text. Leave it empty when showing exact datetimes.

Where each format is used

Surface Format Rationale
Activity feed items Relative (+ tooltip) Feed is scanned for recency; exact time on demand
Chat messages (current session) Exact time — 2:14 pm Within a dated session; specific minute matters
Chat messages (older) Exact date + time — 12 May 2026, 2:14 pm Full context needed; no tooltip required
Investor dashboard card (last activity) Relative (+ tooltip) Recency is the primary signal for engagement
Document updated date Short date — 12 May 2026 Date-level precision sufficient
Notification badges Relative Scanned at a glance

Copy conventions

  • Use 12-hour time with lowercase am/pm: 2:14 pm not 14:14 or 2:14 PM
  • Use abbreviated month: 12 May 2026 not 12/05/2026 or May 12, 2026
  • Separator between date and time: comma + space — 12 May 2026, 2:14 pm
  • "Just now" not "now" or "moments ago"
  • Use plain numerals — no ordinals (12 not 12th)
