Responsive design

Guidelines for making layouts work across screen sizes.

Breakpoints

Name Width Context
Mobile < 768px Single-column layout, stacked cards
Tablet 768px - 1023px Simplified grid, collapsible sidebar
Desktop >= 1024px Full layout with sidebar and multi-column grids

Component-level responsiveness — prefer container queries

For components that need to adapt visually based on the space their parent gives them (not the viewport), use a container query on :host, not a media query.

:host {
  display: block;
  container-type: inline-size;
}

/* Narrow container — stack rather than sit side-by-side */
@container (max-width: 240px) {
  .alert {
    flex-direction: column;
  }
}

Why: a component placed inside a narrow sidebar behaves differently to the same component placed in a wide main column. Media queries only know the viewport — they can't tell the sidebar from the main. Container queries let the component respond to its real, local context without any props or viewport checks.

First used by: la-alert — stacks icon above body below 240px container width. Pattern to follow for any future component whose layout should adapt to local container size (cards, widgets, list items).

Don't reach for it when the breakpoint is about the whole page layout (sidebar collapse, dashboard reflow) — that's a media-query concern and lives at the layout level, not the component.

Grid patterns

  • Use auto-fill / auto-fit with minmax() for card grids — never fixed column counts.
  • Cards reflow naturally as the viewport narrows.
display: grid;
grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
gap: 14px;
  • Full-width components (lists, panels) span all columns with grid-column: 1 / -1.

Layout shifts at breakpoints

Dashboard

  • Desktop (>= 1024px): flex row — sidebar + main content side by side
  • Tablet / mobile (< 768px): column stack — sidebar collapses, content fills full width

Two-column layouts (e.g. data room)

  • Desktop: grid-template-columns: 1fr 1fr
  • Below 800px: single column — panels stack vertically

Sidebar

  • Desktop: always visible, fixed width
  • Tablet: collapsible, toggled via hamburger icon
  • Mobile: hidden by default, slides in as overlay

Touch targets

  • Minimum 44px height and width for all interactive elements on touch devices.
  • Applies to buttons, links, list items, checkboxes, and any tappable surface.
  • Spacing between adjacent touch targets should prevent accidental taps.

Content rules

  • No horizontal scroll — content must fit viewport width at every breakpoint.
  • Content max-width remains 1280px — centred with margin: 0 auto on wide screens.
  • Images and media scale down with max-width: 100%.
  • Tables use horizontal scroll within their container if they overflow, but the page itself never scrolls horizontally.

Test widths

Always check layouts at these widths:

Width Represents
375px Mobile (iPhone SE / small Android)
768px Tablet (iPad portrait)
1280px Desktop (standard laptop)

Component-specific breakpoints

Occasionally a component needs a sub-breakpoint inside a system band. Log those here so they don't drift.

Component Breakpoint Purpose
la-drawer 480px viewport Right and left drawers become width: 100vw (corner radius collapses to 0). Top/bottom drawers are already full-width at every viewport, so the rule doesn't apply to them.

Rule: introduce a component-specific breakpoint only when the system mobile/tablet/desktop bands don't carry the right distinction. Log the pixel value + purpose here; never leave the literal undocumented in a single component file. See decisions/ for architectural rationale.
