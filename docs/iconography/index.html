Iconography

Library

Phosphor Icons — loaded via the local vendor files.

<!-- Page-level (both weights must be loaded at document level — shadow DOM imports via `iconStyles` do not reach slotted light-DOM icons) -->
<link rel="stylesheet" href="../assets/vendor/phosphor/regular/style.css">
<link rel="stylesheet" href="../assets/vendor/phosphor/fill/style.css">

Usage:

<i class="ph ph-house"></i>           <!-- Regular (default) -->
<i class="ph-fill ph-house"></i>      <!-- Fill (tinted surfaces, active/selected) -->
<i class="ph-bold ph-dots-three"></i> <!-- Bold (only for glyphs too thin at small sizes) -->

Weight rules

State Class When
Default / resting ph Icons at rest — most buttons, nav items at rest, inline content icons
Active / selected ph-fill Toggled on, currently-selected nav item
Tinted surface ph-fill Always — inside la-badge, la-alert, la-toast, and any future status-tinted component. The filled glyph matches the solid tinted surface and reads as one visual block; an outline icon against a tinted background feels visually disconnected
Bold ph-bold Only for glyphs that render too thin at small sizes — currently ph-dots-three

Colour rule: icons always inherit color from their parent — never hardcode an icon colour directly.

Sizing

Context Size Notes
Inline with body text 16px --la-font-size-xl
Button with icon 14px --la-font-size-lg
Badge / label icon 13px --la-font-size-md — slightly larger than the 11px badge label for presence
Empty state 32px Hero-scale

Icons inherit font-size from the parent element. Override only when the icon sits outside a standard text context.

Status & variant icons

Canonical icon per variant across every status-tinted component (la-badge, la-alert, future la-toast). If a component uses a status variant, reach for the icon in this table — don't invent a new one. Always pair with ph-fill on tinted surfaces.

Variant Canonical icon Reasoning
danger ph-warning-octagon Octagon = stop-sign shape; reads as "urgent action required"
warning ph-warning-circle Circle reads softer — "heads up, not critical"
success ph-check-circle Standard across UIs — confirmed / resolved
info ph-info Plain info glyph — "informational / in-progress / awaiting"
neutral ph-minus-circle "Inactive / unavailable / no action" — reach for ph-lock specifically when the meaning is locked (legal area not yet unlocked)
accent context-specific Accent = category, not status. Default to ph-tag when no better glyph fits; override with a specific category glyph (e.g. ph-file-text for doc type, ph-buildings for org type)

Rule: if a new component introduces an icon that isn't in this table or the reference below, log it here before shipping. A drift between components (e.g. one uses ph-warning-circle for danger, another uses ph-warning-octagon) is a design-system bug — every audit catches it.

Form selection

Glyphs that mark a chosen item inside a list / picker / option group. Distinct from the status canon — selection is structural (which item is chosen), status is semantic (success / resolved / done). Reach for the plain ph-check, never ph-check-circle, to keep the two signals separable.

Meaning Icon
Listbox option selected (la-select / la-option) ph-check

Weight: default ph (regular). Inline with the option label; never standalone.

Sort (data tables)

Canonical glyphs for sort affordances in la-data-table and any future sortable data primitive. All three sit in the same header cell — the idle glyph becomes the direction glyph when the column is active.

Meaning Icon
Sortable, not sorted (idle affordance) ph-arrows-down-up
Sorted ascending ph-caret-up
Sorted descending ph-caret-down

Weight: default ph (regular). Do not use ph-fill — sort carets are header hints, not tinted-surface glyphs.

Size: inherits from the header cell (--la-font-size-md in data-table). The icon is aria-hidden="true"; screen readers read the aria-sort attribute on the <th> instead.

Document page — status badge

The document status appears as a la-badge in the header meta row alongside entity, category, and legal area. The Phosphor icon goes in the icon slot. Never a standalone icon or icon button — the label text is required to disambiguate "Analysed" from the resolved-insight ph-check-circle that appears on insight cards.

Status Badge variant Icon Icon weight
Analysed (ready) success ph-check-circle fill
Processing info ph-spinner-gap (CSS spin animation) regular
Expired / Missing signature danger ph-warning-circle fill
Flagged by AI warning ph-warning fill

Note on danger badge icon: ph-warning-circle (circle) is used here instead of the canonical ph-warning-octagon (stop sign) — the la-alert banner below the header already carries the full-weight danger signal; the badge is a softer label. This is an intentional divergence from the status variant canon.

Workflow

The four workflow commands each interact with this file:

  • /build — reads this file during scout; uses canonical icons during build; flags any new icon usage for /ready to log
  • /document — reads this file during scout; icons in docs snapshots and usage examples must match the canonical mapping
  • /audit — Gate 8 checks source and preview icon usage against this file; drift is reported as a standards-compliance violation
  • /ready — the only command that edits this file. When a new icon/meaning emerges, /ready Step 4 ("capture new rules") is where it lands

Treat this file as the single source of truth for icon meaning across the system. If it's stale, fix it before adding new icons that would compound the drift.
