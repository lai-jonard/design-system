la-app-shell

The root layout container for every screen. Positions the sidebar on the left and a content area on the right. All pages render inside this shell.

Layout structure

la-app-shell
├── sidebar slot          ← la-sidebar goes here
└── content area
    ├── page header       ← shown when show-header is set
    │   ├── back button   ← shown when show-back is set
    │   ├── breadcrumbs / page title
    │   └── header-actions slot
    └── body row
        ├── main slot     ← scrollable page content
        └── aside slot    ← optional right panel (show-aside)
  • The sidebar and page header never scroll.
  • The main content area scrolls independently (overflow-y: auto).
  • No horizontal scroll anywhere in the shell.

Slots

Slot Required Description
sidebar Yes (practically) Place <la-sidebar> here
main Yes The page content
header-actions No Right-aligned content in the page header — buttons, processing widgets, etc. Only rendered when show-header is set
aside No Right panel content — widgets, tools, details. Only rendered when show-aside is set

Properties

Property Attribute Type Default Description
pageTitle page-title string '' Text shown in the page header when no breadcrumbs are set
breadcrumbs — BreadcrumbItem[] [] Breadcrumb trail — set via JS property. Last item is the current page (rendered as plain text, not a link)
showBack show-back boolean false Shows a back arrow button at the left of the page header
showHeader show-header boolean false Renders the page header bar. Hidden by default
showAside show-aside boolean false Renders the aside panel slot

BreadcrumbItem

interface BreadcrumbItem {
  label: string;
  href?: string;   // if present, renders as an <a> tag; otherwise a <button>
}

Events

Emitted

Event Detail When
la-back — Back button is clicked
la-breadcrumb { label: string, index: number } A breadcrumb item (not the last) is clicked

Listened for

Event Source Effect
la-toggle-aside Any descendant Toggles the aside panel open/closed

Methods

Method Description
openAside() Programmatically opens the aside overlay. Useful for testing or triggering from external controls

CSS tokens

All tokens are defined in src/styles/tokens.ts and can be overridden per-instance with CSS custom properties.

Page header

Token Default Controls
--la-ph-height 40px Header bar height
--la-ph-padding-left 32px Left padding of the header
--la-ph-padding-right 20px Right padding of the header
--la-ph-bg #FAFAFA Header background colour
--la-ph-title-size 18px Page title font size
--la-ph-title-weight 600 Page title font weight
--la-ph-gap 0px Gap between header and main content

Main content area

Token Default Controls
--la-main-margin 8px Margin around the main content card (and aside card)
--la-main-radius var(--la-radius-md) Border radius of the main content card
--la-main-shadow var(--la-shadow-md) Shadow on the main content card

Flat body chrome (unlock pattern)

For focused flows like the unlock prototypes, the inner body card can be flattened so the panel sits directly on the host's --la-color-bg-subtle. Override the four chrome tokens at the host level:

la-app-shell {
  --la-main-margin: 0;
  --la-main-radius: 0;
  --la-main-shadow: none;
  --la-color-bg-page: transparent;
}

Use this when:

  • the slot content is a single self-contained card (e.g. la-unlock-panel) that brings its own chrome, and
  • the page is a focused single-task flow (no aside, no actions widget, no insights list).

The sidebar and page header stay visible — the override only flattens the inner body card. The .main-content slot wrapper should add its own flex centering (display: flex; align-items: center; justify-content: center; height: 100%) so the slot child lands in the middle of the available content area. Reference: dev/prototypes/founder-flow/05/09/16/20-*-unlock.html and decision 46.

Aside panel

Token Default Controls
--la-aside-width 320px Fixed width of the aside panel (desktop)

Breadcrumb behaviour

Set breadcrumbs via the JS property (not an attribute) since the value is an array:

document.querySelector('la-app-shell').breadcrumbs = [
  { label: 'Dashboard' },
  { label: 'Governance' },
  { label: 'Insights' },
];
  • All items except the last render as interactive links or buttons.
  • Items with href render as <a> tags. Items without href dispatch a la-breadcrumb event on click.
  • The last item is always plain text (current page — no interaction).
  • When breadcrumbs is empty, pageTitle is shown instead.

Aside panel — responsive behaviour

The aside panel behaves differently depending on the available content width.

Desktop (content area > 1023px)

  • Aside renders inline, side-by-side with main content.
  • No toggle or close button needed.

Tablet (content area ≤ 1023px)

  • Aside becomes an overlay panel that slides in from the right.
  • A backdrop appears behind it.
  • A close button (×) appears at the top of the aside.
  • Clicking the backdrop closes the aside.
  • Any child component can trigger the toggle by dispatching la-toggle-aside (bubbles, composed).

The breakpoint is a CSS container query on the content area, not the viewport — so the layout responds to the actual available space regardless of sidebar state.

Usage

Basic — sidebar and content only

<la-app-shell style="width: 100vw; height: 100vh;">
  <la-sidebar slot="sidebar" ...></la-sidebar>
  <la-governance-page slot="main"></la-governance-page>
</la-app-shell>

With page header and breadcrumbs

<la-app-shell show-header show-back>
  <la-sidebar slot="sidebar" ...></la-sidebar>
  <la-document-page slot="main"></la-document-page>
</la-app-shell>

<script>
  document.querySelector('la-app-shell').breadcrumbs = [
    { label: 'Dashboard' },
    { label: 'Governance' },
    { label: 'Documents' },
    { label: 'Articles of Incorporation' },
  ];
</script>

With aside panel and header actions

<la-app-shell show-header show-aside>
  <la-sidebar slot="sidebar" ...></la-sidebar>
  <la-button slot="header-actions" variant="primary">Export</la-button>
  <la-insights-list slot="main"></la-insights-list>
  <la-ai-chat slot="aside"></la-ai-chat>
</la-app-shell>

Notes

  • la-app-shell needs an explicit height to fill the viewport — it is display: flex; height: 100%. Typically the parent gives it height: 100vh or it sits inside a full-height wrapper.
  • In docs pages, the shell is rendered inside a fixed-height .shell-frame div — this is just for the docs demo, not how it would be used in production.
  • DialKit can be loaded manually on any docs page by adding <script src="../dialkit.js"></script> — it is not auto-injected.
