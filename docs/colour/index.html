Colour

Token system

All colours are defined as CSS custom properties in src/styles/tokens.ts — the source of truth. Two :root mirrors for light-DOM pages (dev/la-tokens-root.css, docs-src/tokens.css) are generated from it on every build; never edit them by hand. Never hardcode a colour in a component or screen — always reference a token.

Two layers: core primitives (--la-red-500, --la-indigo-50, …) are the only place a colour value lives; semantic tokens (everything below) are var() aliases onto them. Always reach for the semantic name in components.

Colour model: OKLCH

Colour tokens are authored in oklch() (perceptually uniform: equal L = equal brightness, stable hue across a ramp). Each token carries its original hex as an inline comment for Figma cross-reference — oklch(…); /* #HEX · … */. New colour values should be written in oklch. Effect alphas (shadow rgba, scrim, focus ring) stay as rgba() for now. See decision 84.

Contrast: check with APCA (Lc ≥ 75 body text, ≥ 60 non-body). Fix a failing pair by adjusting lightness (L) only — chroma has negligible effect on contrast.

Primary vs neutral vs accent

Three "dark CTA" eras, all still in the file for a reason:

  • --la-color-primary (#18181B) is the near-black primary action — it marks "the next obvious step".
  • --la-color-neutral (#374151) is the softer dark slate, used where primary's near-black feels too heavy or where two solid CTAs sit side by side. (Historically it replaced an even harsher #121212 — too blunt for a product that aims for calm and trust.)
  • --la-color-accent (#405CD0) is the brand indigo for links and attention moments alongside the near-black primary.

Hover states on ghost/tinted buttons

Ghost buttons (danger, success variants) use their semantic surface colour on hover, paired with their semantic border:

  • Danger hover: background --la-color-surface-danger + border --la-color-border-danger
  • Success hover: background --la-color-surface-success + border --la-color-border-success

Why: the hover state confirms the semantic intent before the user commits. The surface tint + coloured border creates a clear "you are about to do something [dangerous/positive]" signal.

Interactive-only rule

--la-color-primary and --la-color-accent appear on interactive elements only — buttons, active states, links, tab underlines — never as background fills, illustration colours, or decorative accents. This keeps the next actionable step visually unmistakable.

Disabled state

All interactive elements at disabled state use opacity: 0.4 on the element itself — no separate disabled token. This keeps the colour intent visible while clearly communicating unavailability.
