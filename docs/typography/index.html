Typography

Typeface

Inter — loaded from Google Fonts at weights 400, 500, 600.

Why Inter: humanist sans that reads well at small sizes. Essential for dense legal data, long description text, and compact status labels.

Weight scale

Weight Value Use
Regular 400 Body copy, descriptions, table cell content
Medium 500 Labels, secondary headings, nav items, badge text
Semibold 600 Page titles, card headings, section headers, CTAs

Rule: headings are semibold, not bold

700 weight at 16–22px reads as heavy and aggressive in a legal product where trust and calm matter. 600 gives structure and hierarchy without visual shouting.

Exception: <strong> tags inside notification bars and alert copy — inline emphasis within body text.

Size scale

The base body size is --la-font-size-base — see decision 15. --la-font-size-lg is a deliberate step up for labels and prominent copy that need more presence than base; never the default. Numeric values live in src/styles/tokens.ts — this table only describes which token plays which role.

Usage Size token Weight Colour token
Page / hero heading --la-font-size-4xl 600 --la-color-text
Section heading --la-font-size-3xl / -2xl 600 --la-color-text
Card / group heading --la-font-size-xl / -lg 600 --la-color-text
Body (primary) --la-font-size-base 400 --la-color-text
Body (step-up) --la-font-size-lg 400 --la-color-text
Label / badge text --la-font-size-base 500 varies
Status / micro label --la-font-size-sm 500 varies
Caption / meta --la-font-size-sm 400 --la-color-text-muted

Text colour tokens

Token Use
--la-color-text Primary readable text, headings
--la-color-text-muted Secondary / supporting copy, subtitles
--la-color-text-faint Muted — captions, dates, disabled labels
--la-color-text-disabled Disabled state text

Never use raw hex for text colours — always reference a token.

Line height

  • Body copy: 1.5
  • Single-line labels and badges: normal
  • Legal / document text (if used): 1.7 — generous for readability

Patterns

Status badges

--la-font-size-sm × --la-font-weight-medium, sentence case. Never all-caps. Keep copy to 1–2 words.

Card headings

--la-font-size-lg × --la-font-weight-semibold × --la-color-text. No uppercase.

Meta / timestamp

--la-font-size-sm × --la-font-weight-normal × --la-color-text-muted. Appears below or beside the primary label.

Links / anchors

Always use the foundation anchor style from la-base-root.css — never write custom colour, text-decoration, or font-weight rules on <a> elements.

  • Colour: --la-color-text-secondary at rest
  • Decoration: underline, text-decoration-color: --la-color-border, thickness: 1px, offset: 2px
  • Hover: decoration colour shifts to --la-color-primary
  • Weight: --la-font-weight-normal

For inline links that need layout overrides (e.g. text-align: right, display: block), add only structural rules to the class — never visual ones. Example:

/* ✅ layout only — visual treatment comes from la-base-root.css */
.forgot { display: block; text-align: right; }

/* ❌ don't do this */
.forgot { color: var(--la-color-primary); text-decoration: none; font-weight: var(--la-font-weight-medium); }

In light-DOM pages, link la-base-root.css after la-tokens-root.css and the styles apply automatically to all <a> elements.

Text wrapping

Applied platform-wide from the base styles — you rarely need to set these yourself.

  • Headings use text-wrap: balance — even line lengths, no orphaned last word. Balance only runs on short blocks (≤6 lines in Chromium), which headings always satisfy. Set on h1–h6 in la-base-root.css.
  • Body text uses text-wrap: pretty — prevents a single dangling word on the last line, at any length. Set on p, li, blockquote, figcaption, dd in la-base-root.css, and inherited across shadow roots via text-wrap: pretty on the :host base in tokens.ts.
  • Don't apply balance to long paragraphs (silently ignored) or to code/pre blocks. pretty is the safe default for short-to-medium copy; leave 10+-line text on the browser default.

See decision 83.
