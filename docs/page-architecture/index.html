Page architecture

How pages are structured in apps built with this design system.

App shell

The top-level layout is provided by la-app-shell. It contains two zones:

[app-shell]
├── [sidebar]         fixed, always visible, collapsible
└── [content area]
    ├── [page header]   fixed at top of content area
    └── [main]          scrollable content below header
  • The sidebar and page header stay fixed — they never scroll.
  • The main content area scrolls independently.
  • No horizontal scroll — content must fit within the viewport width.

Sidebar (la-sidebar)

  • Always present on desktop. Collapsible to icon-only state.
  • Shows the active legal area with a highlighted background.
  • Unlocked areas display sub-tabs (insights, documents, actions, etc.).
  • Locked areas show as dimmed with a lock icon — still visible, not hidden.

Page header

The header sits at the top of the content area and contains:

  1. Breadcrumbs — a clickable trail showing the user's location. Every item is a link except the last (current page).
  2. Back button — optional, shown when the page has a clear parent to return to.
  3. Header actions slot — right-aligned area for page-level actions (e.g. "Export", "Add document").

Breadcrumb pattern

Dashboard / Governance / Insights
  • "Dashboard" and "Governance" are links.
  • "Insights" is plain text (current page, not clickable).
  • Use / as the visual separator.

Page hierarchy

The navigation follows a consistent depth pattern:

Dashboard
└── Legal area (e.g. Governance)
    └── Sub-tab (e.g. Insights, Documents, Actions)
        └── Detail view (e.g. single insight, single document)
  • Dashboard — overview of all legal areas, readiness, deadlines
  • Legal area pages — tabbed view for a single area's content
  • Sub-tabs — insights, documents, actions within a legal area
  • Detail views — focused view of a single item (modal or full page)

Scroll behaviour

  • Sidebar: fixed, does not scroll with content. If sidebar content overflows, it scrolls internally.
  • Page header: fixed at top of content area, always visible.
  • Main content: scrolls vertically. This is the only scroll context the user interacts with day-to-day.
  • Modals: scroll internally with max-height: 90vh and overflow-y: auto. Background content does not scroll while a modal is open.
