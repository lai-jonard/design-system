la-document-page

Full-screen document detail surface. Two-panel layout: detail pane left (name, entity, About section, linked insights, actions), document preview right. The About section renders key terms (flat list for one-per-company docs) or obligations (party-grouped list for bilateral contracts) depending on the document type — the backend determines which renderer applies. Sprint 6 — Stage 1.

Anatomy

The component fills its host element. Left detail pane scrolls independently; right preview pane is fixed within the viewport height.

Anatomy regions
RegionSourceRole
Detail pane.detail-paneLeft column (1fr). Scrollable. Contains all document metadata, About content, linked insights, and actions.
Document header.doc-headerDocument title (documentName) with a status badge, entity name, and meta text (category · legalArea) in a single row below. Replace/Delete actions menu and status badge sit in the top-right. When replacing is true the menu swaps for an "Uploading…" indicator.
Flag banner.flag-wrap + la-alertDanger or warning la-alert shown for flagged-expired, flagged-unsigned, and flagged-ai statuses. Carries icon, title, and the flagReason copy.
About section.detail-section (About)Backend-determined renderer: key-terms renders a flat sentence list; obligations renders party-grouped obligation bullets; loading shows a spinner while analysis runs; empty string hides the section entirely.
Details section.detail-section (Details)Key–value rows for uploadedDate, fileSize, category, and legalArea. Hidden when all four are absent.
Linked insights.detail-section (Linked insights)Clickable severity-icon rows for each LinkedInsight in the insights array. Section is hidden entirely when the array is empty. Consumer wires clicks to open la-insight-modal.
Preview pane.preview-paneRight column (1fr). Subtle surface. Shows a document-file placeholder for ready and flagged statuses; animated spinner for processing.

la-document-page

Page component (src/pages/la-document-page.ts). Two-panel full-screen layout for a single document. Not subject to the /ready promotion gate — this docs draft is the canonical reference. Sprint 6 — Stage 1.

Properties

PropertyAttributeTypeDefaultDescription
documentName document-name string "Untitled document" Display name rendered in the document header.
documentStatus document-status 'ready' | 'processing' | 'flagged-expired' | 'flagged-unsigned' | 'flagged-ai' 'ready' Drives the status badge, flag banner, and preview pane state. processing shows a spinner in the preview pane; all flagged values show the flag banner in the detail pane.
entity entity string '' Optional person or entity associated with this document (e.g. "Jane Lee"). Shown as a sub-line under the document name. Non-interactive in Sprint 6 — People page navigation wires in Sprint 8.
deletable deletable boolean false Backend-gated. When true, the Delete item is rendered in the actions menu.
entityClickable entity-clickable boolean false When true, the entity name in the header renders as an underlined link and fires la-entity-click on click. Use for Employment documents where the entity links back to the People page.
replacing replacing boolean false When true, the actions menu is replaced with an "Uploading…" spinner indicator, preventing further Replace or Delete actions while the upload is in progress.
flagReason flag-reason string '' Supplementary explanation inside the flag banner. Falls back to a generic default for each status type when empty.
category category string '' Document category shown as a meta chip and in the Details section.
legalArea legal-area string '' Legal area name shown as a meta chip and in the Details section.
uploadedDate uploaded-date string '' Human-readable date string shown in the Details section.
fileSize file-size string '' Human-readable file size string (e.g. "142 KB"). Shown in the Details section and in the preview pane sub-line.
insights — LinkedInsight[] [] Linked insights for this document. Each item: { title: string; severity: 'urgent' | 'proactive' }. Consumer wires clicks on insight rows to open la-insight-modal.
aboutType about-type 'key-terms' | 'obligations' | 'loading' | '' '' Selects the About renderer. Backend-determined — no frontend logic decides which applies. Empty string hides the About section entirely.
aboutSummary about-summary string '' One-to-two sentence summary shown above the key terms or obligations list.
aboutItems — AboutItem[] [] Array of items for the About renderer. Key terms: { text: string }[]. Obligations: { party: string; items: string[] }[]. Mix of types is not supported — use the type that matches aboutType.

Slots

No slots. All content is driven by properties.

Events

EventDetailWhen fired
la-insight-select { title: string; severity: 'urgent' | 'proactive' } User clicks an insight row in the Linked insights section. Consumer opens la-insight-modal.
la-replace none User selects "Replace document" from the actions menu. Consumer opens a Replace modal and sets replacing = true during upload.
la-delete none User selects "Delete document" from the actions menu. Consumer opens a Delete confirmation modal.
la-entity-click { entity: string } User clicks the entity name when entityClickable is true. Consumer navigates to the associated entity page (e.g. People page).

All events bubble and are composed — they cross the shadow boundary and can be caught on any ancestor element in the light DOM.

Usage examples

Render inside la-app-shell — the shell provides the breadcrumbs, header, and sidebar. The page fills the slot="main" slot. No aside slot needed.

<la-app-shell show-header page-title="Articles of Incorporation">
  <la-sidebar slot="sidebar"></la-sidebar>
  <la-document-page
    id="page"
    slot="main"
    style="display:block;height:100%;min-height:0;"
  ></la-document-page>
</la-app-shell>

<script>
  var page = document.getElementById('page');
  page.documentName = 'Articles of Incorporation';
  page.documentStatus = 'ready';
  page.category = 'Incorporation';
  page.legalArea = 'Governance';
  page.uploadedDate = '12 Jan 2024';
  page.fileSize = '142 KB';
  page.aboutType = 'key-terms';
  page.aboutSummary = 'The founding document that establishes Acme Corp. as a Delaware C-corporation.';
  page.aboutItems = [
    { text: 'Incorporated in Delaware on 12 January 2024.' },
    { text: 'Authorised share capital: 10,000,000 common shares at $0.0001 par value.' },
  ];
</script>

Wire actions and entity link:

// Replace flow — set replacing while upload is in progress
page.addEventListener('la-replace', function() {
  page.replacing = true;
  // ... upload logic ...
  page.replacing = false;
});

// Delete flow
page.addEventListener('la-delete', function() {
  // open confirmation modal
});

// Entity link (Sprint 8 — Employment documents only)
page.entityClickable = true;
page.addEventListener('la-entity-click', function(e) {
  navigateTo('people.html'); // e.detail.entity contains the name
});

Wire insight rows to open la-insight-modal:

page.addEventListener('la-insight-select', function(e) {
  modal.headline = e.detail.title;
  modal.severity = e.detail.severity;
  modal.open = true;
});

Notes

  • Page, not component · lives at src/pages/la-document-page.ts. Not subject to the /ready promotion gate; this draft is the canonical reference and stays at dev/docs-drafts/.
  • Two About renderers, backend-selected · key-terms for one-per-company documents (Articles of Incorporation, Board Resolutions); obligations for bilateral contracts (Employment Contracts, IP Assignments). The renderer is set by the backend document-type classification — no frontend logic decides which to use.
  • Entity is plain meta text · renders as muted text in the header meta row (e.g. "Jane Lee · Employment"). When entityClickable is true it becomes an underlined primary-colour link and fires la-entity-click. First wired in Sprint 8 for the Employment flow.
  • deletable is backend-gated · set this prop only when the backend confirms the document can be deleted. The prop gates the Delete item in the actions menu; the consumer handles the confirmation modal and post-delete redirect.
  • Actions are in the header menu · Replace and Delete are in a la-menu next to the status badge, not a bottom section. Set replacing = true while an upload is in progress to disable the menu and show an "Uploading…" indicator.
  • Linked insights section hidden when empty · the section is not rendered at all when insights is an empty array. No fallback message is shown.
  • Two-column layout is fixed · left detail pane is 600px, right preview pane fills remaining space. Flag banners use la-alert; spinners use ph-spinner-gap with CSS rotation.
  • Preview pane is a placeholder · shows a document-file icon and "View original" button. Actual PDF/DOCX rendering is engineering's concern at integration time.
  • "View original" button — engineering decision needed · the button is intended to open the raw uploaded file (PDF/DOCX) in a new tab or trigger a download, giving users access to the exact file as uploaded. If the preview pane renders the actual file directly, this button becomes redundant and should be removed. Engineering should confirm their preview approach before wiring it up.
