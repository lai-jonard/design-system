LawAdvisor AI — Design System

A Lit-based component library and the shared language behind every LawAdvisor AI product surface. Everything here — primitives, composed components, pages, and the rules that tie them together — is token-driven and designed to read the same way in code and in prose.

What is this

This is the single source of truth for LawAdvisor AI's UI. If a colour, spacing, radius, motion curve, or interaction pattern shows up in a product screen, it starts here. Component source lives under src/; published docs under docs-src/components/ (with in-progress drafts at dev/docs-drafts/); the rules that govern them in CLAUDE.md and design-language/.

The intent is that a designer, engineer, or AI agent can land on any component page, read in under a minute what it does and when to use it, and find the surrounding context (PRD, decisions, readiness) one click away.
