la-select

Custom single-select combobox. Trigger chrome mirrors la-input (height, border, focus ring, error, label, hint) so a select sits naturally in a form row alongside inputs. Slotted <la-option> children carry the values; rich content (icons, multi-line) is allowed inside each option.

Anatomy

A labelled trigger button with a chevron, plus a floating panel that holds the option list. The trigger is the focusable element when closed; opening moves focus into the active option.

The panel duplicates the .t-dropdown motion pattern from motion.css inside the shadow root — page-scope CSS does not penetrate shadow DOM, so the rules are mirrored using the same motion tokens. Reduced-motion users see no panel transition.

Usage examples

Basic — listen for la-change

<la-select label="Jurisdiction" id="jur"> <la-option value="uk">United Kingdom</la-option> <la-option value="au">Australia</la-option> <la-option value="us">United States</la-option> </la-select> <script> document.getElementById('jur') .addEventListener('la-change', (e) => { // e.detail.value is the picked option's value console.log('jurisdiction:', e.detail.value); }); </script>

Controlled — consumer owns the value

const sel = document.getElementById('jur'); sel.value = state.jurisdiction; sel.addEventListener('la-change', (e) => { // Update state, then assign back — keeps value the source of truth state.jurisdiction = e.detail.value; sel.value = state.jurisdiction; });

In a form row — beside la-input

<div style="display:grid;grid-template-columns:1fr 1fr;gap:16px;"> <la-input label="Matter name"></la-input> <la-select label="Jurisdiction"> <la-option value="uk">United Kingdom</la-option> <la-option value="au">Australia</la-option> </la-select> </div>

Trigger height matches la-input — they sit on the same baseline.

Partner: la-option

Slotted child of la-select. The host element is the focusable target — la-select uses managed focus (rather than aria-activedescendant) so the active option is also the DOM-focused element. tabindex=-1 keeps options out of the document tab order; la-select calls .focus() programmatically as the user navigates.

Slot

SlotAcceptsNotes
(default)Inline contentPlain text or rich content (icons, multi-line). The plain-text fallback is used for type-ahead matching via textContent.
Events. la-option emits no events of its own. Selection always travels through the parent la-select's la-change.
Selection glyph. The check inline with the option label is ph-check (plain check, regular weight). Canon per iconography.md § Form selection — distinct from ph-check-circle which is reserved for the success status. See decision 59.

Notes

Why custom, not native <select>. Native renders the open dropdown via the OS, which means we can't style options, hover, selected state, or icons inside the panel — every product surface would render differently across macOS / Windows / Linux. Custom is more code but gives us full token control and a consistent feel.
Keyboard. ↑/↓ cycle through options, Home / End jump to first / last enabled, Enter or Space selects, Esc closes (focus returns to the trigger), Tab closes-and-progresses to the next focusable element. Type a printable character to jump to the next option starting with that letter — single-character match only, no buffer in v1.
Positioning. Panel floats below by default. When there's not enough space below (e.g. select sits near the viewport bottom), it flips above. Panel uses position: absolute inside the host's shadow DOM — an ancestor with overflow: hidden will clip it. v2 candidate: switch to the Popover API for top-layer rendering when an overflow-clip case appears.
Out of scope (deferred to v2 with a solve). Multi-select, search input inside the panel, ElementInternals form-association.
No exposed part. The trigger and panel are not addressable via ::part. Theme by overriding the documented tokens on the host or an ancestor.
la-select vs filter-menu. Reach for la-select when picking one value in a form row. Reach for a filter-menu (overlay of la-checkbox) when picking multiple values from a toolbar — see the filter-menu snapshot inside la-data-table.
