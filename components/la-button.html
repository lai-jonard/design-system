The primary interactive element across the design system. Wraps a native button with a consistent set of variants and sizes.

Usage examples

Basic

<la-button variant="primary"> Save changes </la-button>

Small — header actions

<la-button variant="secondary" size="sm">Export</la-button> <la-button variant="primary" size="sm">Add item</la-button>

Full width — form CTA

<la-button variant="primary" block > Continue </la-button>

Click handler

const btn = document.querySelector('la-button'); btn.addEventListener('click', () => { // handle action });

Danger — destructive action

<la-button variant="danger"> Delete document </la-button>

CSS part override

/* Target the inner <button> via part */ la-button::part(button) { letter-spacing: 0.3px; }

Icon left — leading glyph

<la-button variant="primary"> <i slot="icon-left" class="ph ph-plus"></i> Add document </la-button>

Icon right — directional hint

<la-button variant="primary"> Continue <i slot="icon-right" class="ph ph-arrow-right"></i> </la-button>

Icon left — complex glyph

<la-button variant="secondary"> <i slot="icon-left" class="ph ph-key"></i> Link API key </la-button>

Notes

Native semantics — The inner <button> element preserves native behaviour: keyboard focus, Enter/Space activation, and form submission. Passing disabled on la-button reflects to the host attribute (which drives the :host([disabled]) dim styling) and forwards to the inner <button> via attribute binding — so keyboard activation is blocked too, not just clicks.
Primary = action — Per the design language, variant="primary" uses --la-color-primary (#18181B). Use it only on the single next actionable step. Never use it decoratively.
Ghost for low-prominence actions — Use variant="ghost" for tertiary actions: "View details", "Back", "Learn more". Resting state is subtle secondary-coloured text with transparent background; on hover it gains a pale-grey fill and darker text. Padding matches the default button so there's no layout shift on hover.
danger vs destructive — variant="danger" is a solid red-filled button for high-visibility system alerts or irreversible one-shot actions. variant="destructive" uses the secondary surface (same background, border, and shadow) but with --la-color-danger-text for label and icon — quieter at rest, red tint on hover. Use destructive as the final commit button in confirmation modals (Delete, Remove) to keep the modal feeling calm without losing the destructive signal.
Font inheritance — la-button uses font-family: inherit on the inner button, so it picks up the host document's typeface. No extra font setup needed.
