la-unlock-panel

Typeform-style question runner for the unlock flow. The user answers one question at a time — sometimes a text field, sometimes a file upload, sometimes a choice — and the panel composes the system primitives for each. One steps array carries everything; one la-unlock-action event carries every interaction. Six panel states cover the lifecycle: pre-start · active · processing · unlocked · next-prompt · dismissed.

Anatomy

The active state lays the question out top-down with generous breathing room. A progress block (counter + stepper bar paired together) sits at the top; the question is the centerpiece beneath it; the field block follows; actions right-align at the bottom. Pre-start carries its CTA at the bottom under the prep list; the active state's Submit button anchors the bottom of the panel.

Composes

The panel does not own input chrome. Every field type maps to a system primitive — when those primitives evolve, this panel inherits the changes for free.

Field type → primitive map
Field typeRenders asCaptures via
'text' · 'email' · 'url' · 'tel' · 'number'la-input with matching typela-change event with {value}
'file'la-upload-boxla-change with {file} · la-remove
'choice'Vertical stack of la-checklist variant="contained"Click handler with single-select enforced internally
'service'la-button variant="primary" size="lg" + manual fallback linkClick on Connect / manual link

Surface chrome composes la-panel for pre-start, active, processing, and next-prompt — a white card on the page subtle bg. The success state is intentionally card-less: a centred ph-fill ph-check-circle in --la-color-success over a headline and sub-line, sitting directly on the subtle bg to contrast with the boxed flow. Step-level errors compose la-alert variant="danger". The progress bar is la-stepper driven by steps.

Usage examples

Define a step set

panel.steps = [ { id: 'cert', title: 'Upload your certificate of incorporation', description: 'PDF, DOC, or DOCX.', fields: [{ type: 'file', name: 'cert', accept: '.pdf,.doc,.docx', required: true, }], status: 'active', }, { id: 'raising', title: 'Are you currently fundraising?', fields: [{ type: 'choice', name: 'raising', required: true, options: [ { value: 'yes', label: 'Yes, actively' }, { value: 'soon', label: 'In the next 6 months' }, { value: 'no', label: 'Not right now' }, ], }], status: 'pending', }, ];

Handle the single event

panel.addEventListener('la-unlock-action', (e) => { const { action, stepId, values } = e.detail; switch (action) { case 'start': panel.unlockState = 'active'; break; case 'submit': submitStep(stepId, values).then(advance); break; case 'skip': markSkipped(stepId); advance(); break; case 'previous': // On step 1, go back to pre-start; otherwise rewind one step. isFirstStep(stepId) ? showPreStart() : rewindOneStep(stepId); break; case 'service-connect': kickOAuth(e.detail.provider); break; case 'service-manual': openManualForm(stepId); break; } });

Multi-field form step

{ id: 'agent', title: 'Add your registered agent', fields: [ { type: 'text', name: 'name', label: 'Agent name', required: true }, { type: 'text', name: 'state', label: 'State', required: true }, { type: 'email', name: 'email', label: 'Email' }, ], status: 'active', }

Surface a backend rejection

// Set step.error to render la-alert[danger] above the field panel.steps = panel.steps.map((s) => s.id === failedStepId ? { ...s, status: 'active', error: 'This document was unreadable. Try a clearer scan.' } : s );

Notes

One question, full focus. Typeform-style. The active step's title is the centerpiece (20px / 600); everything else (counter, stepper) sits quietly around it. This is the rhythm that distinguishes the unlock flow from a generic form.
Auto-advance on single-choice steps. When the active step has exactly one field of type: 'choice', clicking an option immediately fires 'submit'. No Submit button — the click is the commit. For multi-field steps or non-choice fields, Submit is required.
Step-level errors live on the step. Set step.error to render an la-alert variant="danger" above the field block. There's no top-level "failed" state; failure is per-step state. Field values are preserved so the user can fix and resubmit.
Single event, eight actions. Every interaction routes through la-unlock-action. The consumer never wires per-button handlers — switch on e.detail.action and you have everything.
Backward navigation via the back-arrow icon button. Reverses the original Sprint 4 "no Previous button" decision (which scoped backward navigation to stepper-bar click-to-jump). The icon sits inline with Skip + Submit, pushed to the left edge, and emits 'previous' with {stepId} of the step being left. Always enabled in active state — clicking back on step 1 should send the consumer back to pre-start, not nowhere. Bar click-to-jump (when wired on la-stepper) and the back-arrow are now both backward paths; the consumer decides whether to enable both or scope to the arrow.
State derives from steps. The active step is steps.find(s => s.status === 'active'); the stepper bar is derived from steps.map(s => ({label: s.id, status: s.status})). The consumer never juggles two arrays.
Choice today, radio-group tomorrow. Choice fields render with la-checklist[contained] tiles + single-select enforced by the panel. When la-radio-group ships, swap the inner render — the public API of la-unlock-panel doesn't change.
File steps own their own remove path. la-upload-box handles file removal internally; the panel listens for la-remove to clear the captured value. The "I don't have these" fallback is folded into 'skip' — semantically the same outcome.
Per-area step content arrives in Sprints 7–10. This component ships the framework; the actual questions, validation rules, and backend endpoints for Governance / Employment / IP / Fundraising L1 are wired in subsequent sprints.
Success state is card-less. Pre-start, active, processing, and next-prompt all sit inside an la-panel white card on the page subtle bg. The unlocked state breaks that frame — large centred icon-on-top, no card chrome — to read as a deliberate celebration moment, not just another step.
