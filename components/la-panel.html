la-panel

A generic content container with a subtle background, used for secondary information blocks within a layout. Less prominent than a card — sits inside a page rather than floating above it.

Usage examples

Basic

<la-panel> Secondary content here </la-panel>

With heading

<la-panel heading="Key dates"> Completion: 30 June 2026 </la-panel>

Dimmed — locked state

<la-panel heading="Restricted area" dimmed> Content visible but de-emphasised </la-panel>

Clickable — navigable panel

const panel = document.querySelector('la-panel'); panel.addEventListener('click', () => { // handle navigation });

Two-column aside

<div style="display:grid;grid-template-columns:1fr 1fr;gap:12px"> <la-panel heading="Parties">...</la-panel> <la-panel heading="Key dates">...</la-panel> </div>

CSS part override

/* Target the inner wrapper via part */ la-panel::part(panel) { border: 1px solid var(--la-color-border-light); }

Notes

Panel vs card — la-panel uses --la-color-bg-subtle and no shadow, so it sits flush inside a layout. Use it for secondary information blocks (aside columns, metadata groups). Reserve a card surface (--la-color-bg + shadow) for primary content that floats above the page.
Clickable does not imply semantics — The clickable prop adds only a pointer cursor. If the panel needs to be keyboard-navigable or accessible as a link, wrap the content in a native <button> or <a> element instead.
Heading is a prop, not a slot — Headings are passed via the heading attribute rather than a named slot. This keeps the heading style consistent and avoids layout fragility from slotted block elements.
