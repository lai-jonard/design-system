la-stepper

Quiet stepped progress indicator for linear flows. A horizontal bar segmented by step that fills with primary colour left-to-right as steps are completed; skipped steps appear as visible grey gaps in the primary fill. The finished bar — every segment primary — is the completion signal. No per-step animation, no labels — the surrounding flow names each step in its own copy.

Anatomy

A single horizontal row of equal-width 32px × 2px pills with 1px corner radius. One segment per step, separated by an --la-space-xs gap. Shape stays flat; only background colour encodes status.

Anatomy — a 5-step bar mid-flow (2 complete, 1 active, 2 pending)

la-stepper

Quiet stepped progress indicator for linear flows — display-only, drives entirely off the steps property.

Properties

PropertyAttributeTypeDefaultDescription
steps steps StepperStep[] [] Ordered list of steps to render. Each step is a `{ label, status }` object. Each step renders as an equal-width segment in a horizontal bar; the bar fills left-to-right as steps are completed. - `pending` — light rail. The empty track ahead. - `active` — solid primary. Position is the last primary segment. - `complete` — solid primary. Indistinguishable from `active` by design — the transition point shows the user's place. - `skipped` — solid muted. Visible grey gap in the primary fill. - `processing` — deferred; muted-primary tint until consumers wire the document-upload path. The `label` is kept for accessibility (exposed via `aria-label` on each segment) and for future callers that may reintroduce visible text. The component does not render labels visually — the surrounding flow names each step in its own copy.

CSS tokens

Background-only palette. No borders, no green, no amber, no variant-tinted backgrounds — the chrome stays quiet so the surrounding flow can own attention.

Sizing

Token usedRole
--la-space-xsGap between segments

Segment dimensions hardcoded: width: 32px, height: 2px, border-radius: 1px. No tokens exist for these glyph-detail values; matches la-checkbox / la-data-table precedent.

The background-color transition uses --dur-fast × --ease from motion.css. prefers-reduced-motion collapses the transition to instant via an in-source media query.

Usage examples

Basic — assign steps after mount

<la-stepper id="pb"></la-stepper> <script> const pb = document.getElementById('pb'); pb.steps = [ { label: 'Company info', status: 'complete' }, { label: 'People', status: 'active' }, { label: 'Documents', status: 'pending' }, { label: 'Review', status: 'pending' }, ]; </script>

Labels are for screen readers — they're not rendered visibly.

Advance the bar on user action

function markStepComplete(i) { pb.steps = pb.steps.map((s, idx) => ({ ...s, status: idx < i ? 'complete' : idx === i ? 'active' : 'pending', })); }

Skipped — user opted out; flow continues

pb.steps = [ { label: 'Company info', status: 'complete' }, { label: 'People', status: 'skipped' }, { label: 'Documents', status: 'active' }, { label: 'Review', status: 'pending' }, ];

Reassign, don't mutate

// Won't trigger a re-render — mutation pb.steps[0].status = 'complete'; // Triggers re-render — fresh reference pb.steps = [...pb.steps];

Notes

Quiet infrastructure, not a celebration. Per-step micro-rewards dilute the completion moment. This component does the orientation + progress work; the celebration when a flow completes belongs to the consumer (e.g. la-unlock-panel). The bar stays visually calm throughout.
No visible labels. The surrounding flow already tells the user what each step is. Labels live in the aria-label of each circle for screen readers only. Decision traces to solves/260416-2-unlock-onboarding line 37 and solves/260424-1-unlock-flow-progress.
Reassign, don't mutate. steps is a Lit property watched by reference. pb.steps[0].status = '…' won't trigger a re-render — always reassign the whole array.
Segments, not connected line. Each pill reads as its own segment with a gap between — no connecting bar. Loose visual coupling mirrors the fact that steps can be skipped without breaking the flow; the gap is exactly where a skipped segment shows as muted grey within the primary fill.
Fixed-width segments, not flex-fill. Each pill is 32px wide; the bar hugs its content rather than stretching to the container. Keeps the stepper visually compact under the question copy in la-unlock-panel rather than fighting it for width.
Where it's used. Primary consumer is the unlock flow (la-unlock-panel). Option value preserved for future onboarding / KYC / sign-up flows that share the linear-step shape.
Open follow-ups. Backtracking affordance ("go back to step 2") and wiring the processing status for document-upload steps are tracked in solves/260424-1-unlock-flow-progress. The la-progress-bar → la-stepper rename has landed.
