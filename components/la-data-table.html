la-data-table

A data-in table primitive. Takes columns and rows as properties and handles sort, selection, row-click, sticky header, density, empty and loading states. Fully controlled — the parent owns sort, selected, and loading, re-passing them in response to events.

Anatomy

Data-table owns a scrollable wrap, a sortable header, a body of rows, and status-row fallbacks for empty and loading states.

  1. Scroll wrap — bordered container that owns horizontal and vertical overflow. Sticky header pins to its top edge.
  2. Header row — one <th> per column. Sortable headers become clickable and show a sort glyph (ph-caret-up / ph-caret-down / ph-arrows-down-up).
  3. Checkbox column — rendered only when selectable="multi". Header checkbox toggles select-all and shows an indeterminate state when some (not all) rows are selected.
  4. Body row — one <tr> per row. Hover gets a subtle background; selected rows get a stronger active background.
  5. Status row — spans every column and replaces the body when loading is true or rows is empty. Defaults can be overridden via the loading and empty slots.

Usage examples

Columns and rows are properties, not attributes — assign in JS after the element renders.

Basic — columns + rows only

<la-data-table id="docs-table"></la-data-table> <script> const table = document.getElementById('docs-table'); table.columns = [ { id: 'name', label: 'Name' }, { id: 'owner', label: 'Owner' }, { id: 'updated', label: 'Updated' }, ]; table.rows = [ { id: '1', name: 'Contract.pdf', owner: 'Sarah', updated: '2026-04-22' }, { id: '2', name: 'NDA.pdf', owner: 'James', updated: '2026-04-18' }, ]; </script>

Controlled sort

const table = document.getElementById('docs-table'); let sort = { columnId: 'updated', direction: 'desc' }; table.columns = cols.map(c => ({ ...c, sortable: true })); table.rows = applySort(data, sort); table.sort = sort; table.addEventListener('la-sort-change', (e) => { sort = e.detail; table.sort = sort; table.rows = applySort(data, sort); });

Multi-select with bulk actions

<la-data-table selectable="multi" id="docs-table"></la-data-table> <script> const table = document.getElementById('docs-table'); table.columns = cols; table.rows = rows; table.addEventListener('la-selection-change', (e) => { table.selected = e.detail.selected; updateBulkActionBar(e.detail.selected); }); </script>

Custom cell — badge status

table.columns = [ { id: 'name', label: 'Name' }, { id: 'status', label: 'Status', render: (row) => { const badge = document.createElement('la-badge'); badge.setAttribute('variant', row.status); badge.textContent = row.statusLabel; return badge; }, }, ];

Notes

Controlled-only — the parent owns sort, selected, and loading. The component emits events describing the next state; it does not self-mutate. This keeps it predictable under any store, SSR-safe, and trivial to unit-test.
rowKey is required — every row must resolve to a stable string id, either via a keyof T field (default: 'id') or a (row) => string function. Index-based tracking is not supported — selection and sort would desync as soon as rows reorder.
Cell override — return a DOM node or a string — Column.render(row) can return a string, number, Node, or Lit TemplateResult. Vanilla consumers typically use document.createElement; Lit consumers use html\`\` from the library.
Sticky header requires the wrap to scroll — the header pins to the top of the component's own scrollable wrap. If you wrap the data-table inside a separately-scrolling ancestor, set stickyHeader to false, otherwise the header will stick to the inner wrap edge and confuse the user.
Selection models — multi renders a checkbox column and a header select-all with indeterminate state; single highlights one row via click and fires la-selection-change alongside la-row-click; none disables selection entirely. For pure navigation rows without selection, combine selectable="none" with rowClickable.
