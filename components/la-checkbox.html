la-checkbox

A bare form checkbox. Native <input type="checkbox"> wrapped in a tiny Lit element, tokenised to the system palette, with an indeterminate state. Use it for form inputs, filters, bulk actions, and table row selection. Reach for la-choice instead when you need a full row with a label + optional description — la-checkbox is just the square.

Anatomy

A single native input, re-skinned inside the shadow root. The three visual states — unchecked, checked, indeterminate — all share the same box; the glyph inside is drawn with a CSS mask.

  1. Input box — 16px square. Border token --la-color-secondary-border at rest, --la-color-primary when checked or indeterminate.
  2. Checkmark glyph — SVG mask drawn in --la-color-primary-fg, sized via --la-checkbox-icon-size. Rendered only when checked.
  3. Indeterminate bar — a 2px horizontal dash in the primary foreground colour. Rendered when indeterminate — precedes a definite checked/unchecked decision (typical: "some, not all" row selection).
  4. Focus ring — --la-focus-ring-width outline in --la-color-border-focus on keyboard focus (:focus-visible). Matches the rest of the system's focus treatment.

Usage examples

Standalone — aria-label only

<la-checkbox label="Include archived" checked></la-checkbox>

Indeterminate — select-all in a partial selection

<la-checkbox label="Select all rows" .indeterminate=${someSelected} .checked=${allSelected} @la-change=${toggleAll}></la-checkbox>

Inside la-data-table

<la-data-table selectable="multi"> </la-data-table> <!-- la-data-table composes la-checkbox --> <!-- for header + row selection -->

Notes

la-checkbox vs la-choice — la-choice is a statement (the row is the component — onboarding, consent, settings), la-checkbox is a control (the row is owned by something else — table, menu, toolbar, form grid). Full decision table in design-language/components.md.
Indeterminate is a visual state — pair it with a separate checked value when "some, not all" items are selected. On user toggle, indeterminate is cleared and checked is set — the component does not self-manage the tri-state logic; consumers decide whether the next state is "all on" or "all off".
la-change fires on user toggle only — programmatically setting .checked = true does not dispatch an event. This matches the native change convention and prevents feedback loops in controlled patterns.
disabled forwards to the inner input — reflects to the host for the visual dim (opacity: 0.4, pointer-events blocked) and is passed through to the native <input> so Tab and Space/Enter are also blocked, not just clicks.
