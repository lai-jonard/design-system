Usage examples

Basic setup

const widget = document.querySelector('la-actions-widget'); widget.actions = [ { label: 'Add employee', description: 'Register a new employee and set their role.', icon: 'ph ph-user-plus', }, { label: 'Manage people', description: 'Review and update employee records.', icon: 'ph ph-users', }, ];

Listening for la-action

widget.addEventListener('la-action', (e) => { // e.detail is the ActionItem that was clicked console.log(e.detail.label); }); // or listen on any ancestor — event bubbles document.addEventListener('la-action', (e) => { handleAction(e.detail); });

In the aside panel

<!-- Legal area aside --> <aside> <la-actions-widget id="actions"></la-actions-widget> <la-document-list id="docs"></la-document-list> </aside>

Empty state — no actions

// Passing an empty array renders the widget // shell with no cards — pair with la-empty-state // at the consumer level if needed. widget.actions = [];

Notes

JS property only — actions is an array and cannot be set via HTML attribute. Always assign it as a JavaScript property: widget.actions = [...]. Setting actions="..." in HTML will not work.
Only label is required — description and icon are optional. description is stored and included in the la-action event detail but is not rendered visually — each row shows icon + label only.
Icon strings are Phosphor classes — pass the full class string as you would write it in HTML: "ph ph-user-plus", "ph-fill ph-check", or "ph-bold ph-dots-three". The component renders <i class={icon}> inside the shadow DOM where Phosphor CSS is already loaded.
Icon convention — action type, not subject — icons should represent what the user is doing, not the subject matter. Use ph-upload-simple for any upload action, ph-file-arrow-up for filing or submitting a document, ph-arrow-right for transferring or moving something, ph-user-plus for adding a person. Avoid subject-specific icons like ph-trademark or ph-globe — these read as decorative rather than actionable.
Copy rules for actions — labels must be verb-first and sentence case: "Add employee", "Review share structure", "Register trademark". Descriptions should be 5–15 words of plain context — why this action matters now, not what the button does. Follow British English throughout.
Lives in the aside panel — la-actions-widget sits in the slot="aside" of la-app-shell on legal area pages. On desktop it is always visible; on tablet it appears as an overlay drawer. See the aside panel pattern in design-language/components.md.
la-action bubbles and is composed — the event crosses the shadow boundary, so you can listen at the page level or on any ancestor element. Use e.detail.label to route to the correct handler.
