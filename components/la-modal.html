la-modal

Dialog overlay for confirmations, forms, and multi-step flows. Blocks the rest of the page with a backdrop and takes focus until dismissed. Closes on X button or backdrop click.

Anatomy

A centered panel on a backdrop. Three regions: header, body, footer

Heading
Optional description — adds context below the heading.
Body content — default slot
Footer actions — named slot
  1. Heading — the heading prop, semibold 18px.
  2. Description — optional description prop below the heading. Sized and coloured like body copy. Hidden entirely when empty.
  3. Close button — la-icon-button size="sm" with ph-x. Dismisses the modal.
  4. Body — the default slot. Any content: text, forms, lists, nested components. Scrolls internally when content exceeds available height.
  5. Footer — the footer named slot. Auto-hidden when empty via :empty. Separator above only shows when the body is scrolling.

Usage examples

Basic — heading + body

<la-modal heading="Rename document" open> <p>Enter a new name for this document.</p> </la-modal>

Confirmation — destructive pattern

<la-modal heading="Remove employee?" open> <p>This can't be undone.</p> <div slot="footer"> <la-button variant="secondary">Cancel</la-button> <la-button variant="danger">Remove</la-button> </div> </la-modal>

With description — context under heading

<la-modal heading="Remove employee?" description="This can't be undone." open > <p>Any linked documents will be unassigned.</p> <!-- footer --> </la-modal>

With form — intake pattern

<la-modal heading="Add employee" open> <la-input label="Full name"></la-input> <la-input label="Role"></la-input> <div slot="footer"> <la-button variant="secondary">Cancel</la-button> <la-button variant="primary">Add</la-button> </div> </la-modal>

Large — document review

<la-modal heading="Review contract" size="lg" open > <!-- wider content --> </la-modal>

Opening / closing programmatically

const m = document.querySelector('la-modal'); m.open = true; // open m.open = false; // close m.addEventListener('close', () => { // handle dismiss — form reset, navigation, etc. });

Notes

Chrome rules — three sizes: sm 440px (focused confirmations, share actions, short wizard steps), default 560px, lg 640px (document review, wider forms). All three share the same chrome: --la-radius-xl2 (24px) corners, --la-shadow-lg. Header and footer stay pinned; only the body scrolls, capped at 90vh panel height. Backdrop uses --la-color-scrim (rgba(0, 0, 0, 0.2)) — shared across modals and any future overlay that dims the page. From design-language/layout.md Modals section.
Footer separator is conditional — the rule above the footer only appears when the body content is scrollable (a [scrollable] attribute is auto-toggled on the host via a ResizeObserver on the body). Short content = no separator; long content that requires scrolling = separator signals "more content above". Keeps chrome minimal when it isn't load-bearing.
Close triggers — the X button in the header and clicking the backdrop. Both dispatch close and set open = false. For cancel/primary action buttons in the footer, the consumer wires their own close (onclick="…open = false").
Open & close animation — backdrop fades in/out; panel scales between --scale-enter (0.96) and 1. Duration --dur-modal (250ms), curve --ease-out. Both directions driven by CSS transitions on opacity and transform — GPU-safe, no JS. The visibility property flips instantly on open and after the transition on close, so the closed modal doesn't block clicks. Collapses to 0ms under prefers-reduced-motion: reduce.
Accessibility — closed in v1 — the host sets role="dialog" and aria-modal="true" on connect, so screen readers announce the modal as a dialog. Escape closes the modal (global keydown listener, gated on open). Close button uses la-icon-button with an accessible label.
Accessibility — still deferred — no focus trap inside the panel, no return-focus-to-trigger on close. Keyboard users can Tab out of the modal into the page behind it. Fix before using for keyboard-primary flows.
Close icon will change — v1 uses the text character ✕. Per iconography.md the canonical close icon is ph-x. Source drift flagged for /ready.
