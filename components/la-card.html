la-card

A surface container used to group related content. Provides a visual boundary (border + background + radius) without adding layout logic. Takes an optional heading prop for a titled header section.

Properties

PropertyAttributeTypeDefaultDescription
heading heading string '' —
nopadding nopadding boolean false —

CSS tokens

All visual values come from the design token system defined in src/styles/tokens.ts. Override per-instance via style or a wrapping CSS rule.

Usage examples

Basic — heading + body

<la-card heading="Company overview"> A concise summary sits here. </la-card>

Without heading — bare container

<la-card> Content without a title section. </la-card>

Edge-to-edge list — nopadding

<la-card heading="Documents" nopadding> <div class="row">Articles of Association</div> <div class="row">Shareholder agreement</div> </la-card>

Composed — card hosting other primitives

<la-card heading="Actions"> <p>Three items need review.</p> <la-button variant="primary"> Review now </la-button> </la-card>

Notes

Card vs panel — reach for la-card when content needs a bounded, floating surface with a visible border. Reach for la-panel when content should sit inside a layout without floating — a de-emphasised container that recedes into the page.
No layout logic — la-card is purely a surface. Stacking, gaps, and internal alignment are the consumer's responsibility. This keeps the primitive small and composable.
Use nopadding for self-spaced children — when the inner content is a list of rows that each own their padding (e.g. la-list-item, la-document-item), turn off card padding so rows reach the card's edge.
