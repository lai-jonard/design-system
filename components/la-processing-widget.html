la-processing-widget

Keeps users informed while documents are being uploaded and analysed. Floats bottom-right of the content area; renders nothing when items is empty. Composes la-document-item — the widget owns the tray chrome (title, collapse toggle) and dispatches la-reupload / la-dismiss-item; the rows themselves are standard document items.

Anatomy

One surface, header above an animated items wrapper. Per-row actions live in the document item's actions slot, and the dismiss × is built-in via la-document-item's dismissible prop.

Usage examples

Basic — active items

<la-processing-widget></la-processing-widget> const widget = document.querySelector( 'la-processing-widget' ); widget.items = [ { name: 'Contract.pdf', area: 'Employment', status: 'processing' }, ];

Handle events

widget.addEventListener('la-reupload', e => { openUploadFor(e.detail); }); widget.addEventListener('la-dismiss-item', e => { widget.items = widget.items.filter( i => i !== e.detail ); });

Float bottom-right of the content area

<la-app-shell> <la-sidebar slot="sidebar"></la-sidebar> <div slot="main">...</div> </la-app-shell> <div style="position:fixed; bottom:16px; right:16px; width:340px;"> <la-processing-widget></la-processing-widget> </div>

Failed item with error message

widget.items = [{ name: 'Scan-corrupted.pdf', area: 'Employment', status: 'failed', error: 'File is unreadable.', }];

Drive the timeout state

// Consumer-driven threshold (typically ~45s) const t = setTimeout(() => { widget.timeout = true; }, 45000); // Clear when items resolve widget.addEventListener('la-dismiss-item', () => { if (widget.items.length === 0) { clearTimeout(t); widget.timeout = false; } });

Notes

Renders nothing when empty. When items is an empty array the component returns nothing from Lit and produces no DOM. Mount eagerly and populate later — no separate visibility prop needed.
Dismiss is per item, terminal states only. The × appears on completed and failed rows. It is never rendered on processing rows — dismissing mid-process would orphan the eventual outcome (the backend's status broadcast for that document is dropped defensively once dismissed_at is set, per Sprint 1's processing-item ledger).
Re-upload is host-routed. The widget fires la-reupload with the item detail; it does not host an upload surface itself. The host opens the upload component in the row's originating context (the insight modal, add-person modal, etc.), or a minimal modal if that context is gone.
Title is computed. The header title is derived from item counts ("2 processing, 1 completed", "Processing complete", "1 item failed"). Not configurable via a prop.
Internal status mapping. The widget's external API uses processing | completed | failed — natural domain vocab. Internally it maps completed → ready before passing to la-document-item, because "ready" is the right word once the document is viewable in the library. Don't "fix" the mapping — the divergence is intentional.
Collapse toggle is by design. Diverges from Sprint 1 spec (always-expanded). LAI design owns this divergence — the toggle gives hosts UI customisability and aligns with future right-rail panels.
Timeout state has no dismiss button — by design. The widget today has no in-app re-entry surface (no persistent notification, no badge on the legal area). If a user dismissed the alert and walked away, they would have no way to learn when their docs finished. The "you can come back to check" copy plus the persistent widget body is the affordance. Add a dismiss button only when a re-entry surface exists.
Generic-error fallback. When a row is failed and error is empty, the widget supplies "We couldn't process this document. Try uploading it again." to la-document-item. Hosts that have a specific reason should pass it as error; the fallback only fires when nothing better is available.
Per-item dismiss is built-in via la-document-item. The widget passes ?dismissible=${item.status !== 'processing'} to each row. The row renders its own trailing × via la-icon-button and emits la-dismiss; the widget catches that and rebroadcasts as la-dismiss-item. Consumers don't slot anything for dismiss — the row owns the affordance.
Spinner colour is --la-color-primary, not --la-color-processing. Both the header spinner and the in-row processing spinner inside la-document-item use primary (near-black) for a calm monochrome reading on white surfaces. This is a soft tension with CLAUDE.md's "primary on interactive elements only" rule — a spinner is a status indicator, neither interactive nor decorative. The visual call wins; the rule's spirit (don't use primary as brand decoration) is intact.
