Usage examples

Single boolean (default)

<la-choice
  ?checked=${this.agreed}
  @la-change=${(e) => this.agreed = e.detail.checked}
>
  I agree to the terms and conditions
</la-choice>

Single-select group (choice field type) — radio behaviour, parent manages exclusivity

<div role="radiogroup">
  ${options.map(opt => html`
    <la-choice
      variant="contained"
      ?checked=${selected === opt.value}
      @la-change=${(e) => { if (e.detail.checked) selected = opt.value; }}
    >${opt.label}</la-choice>
  `)}
</div>

Multi-select group (multi-choice field type) — each tile toggles independently

<div role="group">
  ${options.map(opt => html`
    <la-choice
      variant="contained"
      ?checked=${selected.includes(opt.value)}
      @la-change=${() => {
        selected = selected.includes(opt.value)
          ? selected.filter(v => v !== opt.value)
          : [...selected, opt.value];
      }}
    >${opt.label}</la-choice>
  `)}
</div>

Notes

Self-managed toggle — Clicking toggles checked internally and fires la-change. For controlled patterns (single-select groups), listen to la-change and set checked from the parent. The two are not in conflict.
Inner checkbox is decorative — la-checkbox is wrapped with pointer-events: none and aria-hidden="true". The host element carries role="checkbox" and aria-checked — it is the only accessible and interactive element in the component.
Invalid is not checked — invalid and checked are independent boolean attributes. A row can be invalid and unchecked (unmet condition), invalid and checked (edge case), or neither. The invalid state signals a problem; it does not block the click toggle.
Group roles — Add role="radiogroup" to the wrapper for single-select groups so screen readers understand mutual exclusivity. Add role="group" for multi-select. The individual tiles always carry role="checkbox" — the parent role contextualises them.
filled modifier — add filled to a contained tile when the checked state should feel like a positive completion rather than a selection. At rest the tile is identical to contained. When checked, the primary border is replaced with a --la-color-surface-success fill and no border — a softer, greener signal used in pre-raise checklists and confirmation steps where each item marks progress done.
la-choice vs la-checkbox — la-choice is a statement (the row is the component — onboarding, consent, unlock steps). la-checkbox is a control (the row is owned by a table, menu, toolbar, or form grid). Full decision table in design-language/components.md.
