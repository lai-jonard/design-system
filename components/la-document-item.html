la-document-item

Atomic document row used across every surface that lists documents — the library, processing tray, and data room. Covers four statuses (none, ready, processing, failed), accepts an actions slot for consumer-supplied buttons, and exposes a built-in trailing dismiss button via dismissible.

Anatomy

One row, four regions: a status icon, the document text, an optional consumer actions slot, and an optional built-in dismiss button. The whole row is one focusable button when interactive; clicks inside the actions slot or on the dismiss button do not fire row selection.

Usage examples

Basic — name only

<la-document-item name="Contract.pdf"></la-document-item>

Ready with association

<la-document-item
  name="Board-Resolution.pdf"
  association="Corp Governance"
  status="ready"></la-document-item>

Failed with error and slotted actions

<la-document-item
  name="Scan-corrupted.pdf"
  status="failed"
  error="File is unreadable. Please upload a clearer copy.">
  <la-button slot="actions" variant="secondary" size="sm">Re-upload</la-button>
</la-document-item>

Dismissible row — built-in trailing X

<la-document-item
  name="Doc.pdf"
  status="ready"
  dismissible
  @la-dismiss="${onDismiss}"></la-document-item>

Dismissible alongside Re-upload — slot left, X right

<la-document-item
  name="Scan-corrupted.pdf"
  status="failed"
  error="File is unreadable. Please upload a clearer copy."
  dismissible
  @la-dismiss="${onDismiss}">
  <la-button slot="actions" variant="secondary" size="sm">Re-upload</la-button>
</la-document-item>

Listen for selection

const item = document.querySelector('la-document-item');

Notes

Actions slot isolation — Clicks inside the actions slot do not fire la-select. The component walks the composed event path and returns early if any node carries slot="actions". Wire your action button handlers independently.
Dismissible suppresses la-select — When the built-in X is clicked, only la-dismiss fires. The handler calls stopPropagation(), and the row's path filter also short-circuits on data-dismiss="true". Slot-isolation and dismiss-isolation compose: a slotted Retry stays independent of both row click and dismiss. See decision 50 for the composition pattern.
Filename truncation — The name value is set as the native title attribute on the name span. Long filenames truncate with ellipsis in narrow containers and reveal the full name on hover.
Processing widget mapping — la-processing-widget uses a completed status in its own API, which it maps to ready before passing to la-document-item. This divergence is intentional — "ready" is the correct word once a document is viewable. Do not normalise it away.
Error message scope — Only rendered when status="failed" and error is non-empty. Setting error on any other status has no visual effect.
Keyboard behaviour — When interactive, the row is a native <button>. Enter and Space fire la-select. The built-in dismiss is its own focusable la-icon-button — Tab moves into and out of it cleanly. Focus rings use :focus-visible and are not suppressed.
ready is transitory — status="ready" is a brief confirmation after processing completes (~3s), after which the parent should revert to none. The component holds no internal timer — the consumer owns the revert. See the Status lifecycle demo above.
Re-upload vs Retry — use Re-upload when the failure means the user has to provide a new file (corrupted scan, wrong format, illegible). Use Retry when the existing file just needs another attempt at the same operation (transient network, parser timeout, server hiccup). The action verb should match what the user actually has to do — anything else is dishonest.
Spinner colour is --la-color-primary. The processing-state spinner uses primary (near-black) for a calm monochrome reading. See decision 53 for the rationale and the soft tension with CLAUDE.md's "primary on interactive only" rule.
box-sizing: border-box on the row — required so width: 100% matches the host width exactly. Without it, the row's padding extended past the host by the padding amount; ancestors with overflow: hidden (la-processing-widget tray) would clip the rightmost contents (the dismiss X). The fix is in source; consumers don't need to do anything.
Open gap — row icon size tokens — File glyph and status badge currently size off --la-font-size-lg by happenstance, and the composed la-spinner uses md (16px). An --la-icon-row-* token family is logged as a backlog item; sizing here will track that family if/when it lands. Not blocking v1.
