Usage examples

Basic — title only

<la-toast id="toast-save" title="Changes saved." ></la-toast> document.querySelector('#toast-save').show();

Title + message

<la-toast id="toast-send" title="Contract sent for signature." message="Anna Chen — Employment Contract 2026" ></la-toast>

With icon slot

<la-toast title="All documents reviewed."> <i slot="icon" class="ph-fill ph-check-circle" style="color:#1F7C51;font-size:18px;" ></i> </la-toast>

Listen for dismiss

const toast = document.querySelector('la-toast'); toast.addEventListener('dismiss', () => { // toast closed — update state if needed });

Custom duration

<la-toast title="Reminder set." duration="5000" ></la-toast>

Notes

Copy pattern — past tense, action-focused — Toast copy should confirm what just happened, not describe what will happen. "Contract sent." not "Sending contract…". "Changes saved." not "Save successful". Keep titles to one short clause.
No self-positioning — la-toast does not portal itself into a fixed overlay. Place it in your layout and position it with CSS — typically position:fixed; bottom:24px; right:24px; on the host element.
Calling show() again resets the timer — if show() is called while the toast is already open, a new dismiss timer starts from zero. The previous timer is not cancelled — both will fire. To safely re-trigger, set open = false first.
Default slot vs message prop — use the message prop for plain text. Use the default slot when you need inline markup — a link, emphasis, or a truncated string you control with CSS. When message is non-empty, the default slot is not rendered.
