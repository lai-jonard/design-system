la-toast-stack

Sonner-style stacked notification manager. Place once in your layout — it establishes the aria-live region and handles positioning, stacking, hover-expand, and auto-dismiss for all active toasts. Use the exported toast() function to trigger toasts from anywhere in the application.

Usage examples

Place once in layout

<!-- Near the top of <body> --> <la-toast-stack></la-toast-stack>

Basic — programmatic

import { toast } from './la-toast-stack.js'; toast('Changes saved.');

With message

toast('Contract sent for signature.', { message: 'Anna Chen — Employment Contract 2026', });

Variant shorthands

toast.success('All documents reviewed.'); toast.warning('Signature pending.'); toast.danger('Upload failed.');

Listen for dismiss

const stack = document.querySelector('la-toast-stack'); stack.addEventListener('la-toast-dismiss', (e) => { console.log('dismissed:', e.detail.id); });

Custom position

<la-toast-stack position="top-right" ></la-toast-stack>

Via element method directly

const stack = document.querySelector('la-toast-stack'); stack.add({ title: 'Changes saved.', variant: 'success', duration: 5000, });

Notes

Place the element for reliable screen reader support — the aria-live="polite" region must be in the DOM before the first toast fires. The toast() function auto-injects an element if none is found, but screen readers may miss the first announcement because the live region wasn't established at page load. Placing <la-toast-stack> explicitly is the recommended pattern.
Hover to expand — at rest, older toasts peek behind the newest (scaled down, staggered upward). Hovering the stack expands all toasts to their full height with spacing between them. Each toast can be read and dismissed individually in the expanded state.
Position changes are live — updating the position attribute or property moves the stack immediately. Any active toasts move with it. Useful when the layout shifts (e.g. a sidebar opens and the bottom-right corner becomes obscured).
Max-visible queuing — toasts beyond maxVisible are not discarded — they are held in the internal list and become visible as older ones are dismissed. The queue drains naturally; no explicit queue management is needed.
Toast heights are measured at runtime — the stack uses a ResizeObserver on the first item to track actual height for the hover-expand offset calculation. The fallback is 88px. No layout measurement is needed before calling add().
