la-tooltip

A hover- or focus-revealed tip that anchors to any element. Use for short supplementary context — the meaning of an icon-only button, format guidance on a field, or a label's expansion. Two appearances — dark (default, inverted surface, for action labels) and light (white surface with subtle border, for status and info hints) — four positions, and size="sm" for compact icon-button labels. Sibling: la-info-hint is a purpose-built composition for inline info glyphs — prefer it over wrapping la-tooltip around a bare icon.

la-tooltip

A hover- or focus-revealed tip that anchors to any element. Dark (default) and light appearances, four positions, and an optional sm size for icon-button labels.

Properties

PropertyAttributeTypeDefaultDescription
tip tip string '' —
position position TooltipPosition 'top' —
appearance appearance TooltipAppearance 'dark' —
size size 'default' | 'sm' 'default' —
maxWidth max-width string '240px' Maximum width of the tooltip bubble. Defaults to `240px` so longer tips wrap onto multiple lines instead of stretching across the viewport. Set to an empty string (`max-width=""`) to opt out and let the tooltip size to its content on a single line.

Usage examples

Dark sm — icon button label

<la-tooltip tip="Show resolved items" size="sm" position="left"> <la-icon-button label="Show resolved items"> <i class="ph ph-check-circle"></i> </la-icon-button> </la-tooltip>

Light — badge status hint

<la-tooltip tip="Deadline in 12 days" appearance="light"> <la-badge variant="danger">Urgent</la-badge> </la-tooltip>

Long tip — wrapping with max-width

<la-tooltip tip="Investors will expect this during due diligence." appearance="light" max-width="200px"> <la-button variant="secondary">Learn more</la-button> </la-tooltip>

Single-line — opt out of wrapping

<la-tooltip tip="Ctrl + K" max-width=""> <la-icon-button label="Open command palette"> <i class="ph ph-magnifying-glass"></i> </la-icon-button> </la-tooltip>

Notes

Dark for actions, light for info — use dark (default) on icon buttons where the tooltip is the primary label. Use appearance="light" on badges, status glyphs, and inline info where the tip is supplementary context on an already-labelled element.
size="sm" for icon button labels — icon-only buttons use size="sm" so the tooltip reads as a compact action label rather than an explanatory paragraph. Default size is for longer contextual tips (field guidance, status expansions, wrapping copy).
Position near viewport edges — the tip is absolutely positioned and centered on the trigger. If the trigger is at the right edge of the layout, use position="left" so the tip extends into the content area. position="top" centered on a right-edge button can extend past the viewport boundary.
Prefer la-info-hint for inline info glyphs — la-info-hint is a purpose-built composition of la-tooltip appearance="light" on an info icon, with three sizes and a proper button role. Use it over wrapping la-tooltip around a bare icon glyph — the info-hint handles keyboard access and sizing correctly.
Known a11y gaps (deferred) — no aria-describedby wired between the trigger and tip; no prefers-reduced-motion override for the show/hide transition. Both are deferred v1 gaps, flagged in components.md.
