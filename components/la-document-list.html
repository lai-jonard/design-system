la-document-list

Data-driven document library component. Accepts a flat array of DocumentData objects, groups them by optional category , and renders each as an la-document-item row inside an la-card .

Usage examples

Basic — flat list

const list = document.querySelector('la-document-list'); list.documents = [ { name: 'Contract.pdf', status: 'ready' }, { name: 'Policy.pdf', status: 'processing' }, ];

Categorised

list.documents = [ { name: 'Contract.pdf', category: 'Employment' }, { name: 'Board-Res.pdf', category: 'Corp Governance' }, { name: 'Policy.pdf', category: 'Employment' }, ];

Listen for row selection

list.addEventListener('la-select', (e) => { // e.target is the la-document-item that was clicked console.log(e.target.getAttribute('name')); });

Custom heading

<la-document-list heading="Related documents" ></la-document-list>

Notes

Array prop — JS only — documents must be set via JavaScript property assignment (el.documents = [...]). HTML attributes only support strings — passing JSON as an attribute is not supported and will not work.
Category grouping — Category labels are shown automatically when two or more distinct non-empty categories exist in the array. A single category or all uncategorised items renders a flat list with no labels.
la-select bubbles — Row selection events bubble up from the composed la-document-item elements. The event target is the individual la-document-item. Use e.target.getAttribute('name') to identify which document was selected.
Per-item actions — la-document-item exposes an actions slot for injecting buttons into each row. la-document-list does not forward this slot — if per-item actions are needed, use la-document-item directly. The Company Manual uses name-only mode; grouped action patterns are deferred.
