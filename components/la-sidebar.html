Usage examples

Basic placement in la-app-shell

<la-app-shell show-header> <la-sidebar slot="sidebar" active-area="corp-gov" unlocked-areas='["corp-gov"]' user-name="Alex Chen" user-role="Acme Corp. · Founder" user-initials="AC" ></la-sidebar> </la-app-shell>

Routing on la-nav-change

const sidebar = document.querySelector('la-sidebar'); sidebar.addEventListener('la-nav-change', (e) => { const { area, subtab } = e.detail; if (subtab === 'company-manual') { router.go('/governance/manual'); } else if (area) { router.go(`/legal-areas/${area}`); } });

Updating unlocked areas dynamically

const sidebar = document.querySelector('la-sidebar'); // After user completes corp-gov onboarding: sidebar.unlockedAreas = [ 'corp-gov', 'employment', ];

Responding to collapse toggle

sidebar.addEventListener( 'la-sidebar-toggle', (e) => { // e.detail.collapsed — true = now collapsed appShell.setAttribute( 'sidebar-collapsed', e.detail.collapsed ); } );

Notes

Fixed nav items — The five areas (Dashboard, Governance, Employment, IP Protection, Fundraising) and their sub-tabs are defined inside the component. They are not slot-configurable. This keeps routing logic predictable and ensures area ids are a stable contract between the sidebar, la-app-shell, and the host application.
Dashboard is always unlocked — Even with an empty unlockedAreas array, the Dashboard item is always accessible. All other areas show a padlock icon and remain clickable when locked — the host should respond to the la-nav-change event with an unlock prompt or upsell screen.
Collapsed state persists automatically — The sidebar writes to localStorage under the key la-sidebar-collapsed whenever the collapse button is toggled. On load, the persisted value overrides whatever is set on the collapsed property. Clear this key to reset to the default.
Collapse animation timing — On collapse, icon positions snap to centre after the label-fade duration (--dur-fast) so the transition appears sequenced (labels fade first, then icons snap). On expand, icons snap immediately so they spread outward as the sidebar widens.
Icon fill convention — Active nav items use the Phosphor fill variant (ph-fill ph-icon-name); resting items use the outline variant. This is handled internally — no configuration needed.
