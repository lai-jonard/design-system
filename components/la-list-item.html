la-list-item

Three-column structural row for lists — icon, content, meta, and a trailing actions slot. Non-interactive by default; flip clickable to turn the row into a native button with hover, focus-visible, and keyboard activation. Clicks inside the actions slot do not bubble to the row — consumers can place per-row action buttons without hijacking the main row click.

Anatomy

Articles of Incorporation Corporate
  1. Row shell — flex container with align-items: center, gap --la-space-sm, padding --la-space-sm × --la-space-md. Renders as <button> when clickable, otherwise <div>.
  2. Icon — leading column. slot="icon". Auto-hides when empty. Typography size --la-font-size-lg, colour --la-color-text-muted.
  3. Content — default slot. Flex-grows to fill available width, wraps when needed.
  4. Meta — trailing text. slot="meta". Smaller (--la-font-size-sm), muted, nowrap.
  5. Actions — trailing button row. slot="actions". Click-stopped at the container — children do not bubble their click to the row.
  6. Separator — 1px bottom border in --la-color-border-light. Auto-removed on :last-child.

Clicks inside the actions slot do not bubble to the row — consumers can place per-row action buttons without hijacking the main row click.

Properties

PropertyAttributeTypeDefaultDescription
error error boolean false Tints the row background with `--la-color-surface-danger`.
clickable clickable boolean false When true, the row renders as a native button — tab-focusable, keyboard-activated, hover + focus styles.

Usage examples

Basic — content only

<la-list-item> Articles of Incorporation </la-list-item>

Icon + content + meta — typical three-column

<la-list-item> <i slot="icon" class="ph ph-file-text"></i> Articles of Incorporation <span slot="meta">Corporate</span> </la-list-item>

With actions — edit + remove icon buttons

<la-list-item> Alex Chen <span slot="meta">CEO</span> <span slot="actions"> <la-icon-button size="sm" label="Edit"> <i class="ph ph-pencil-simple"></i> </la-icon-button> </span> </la-list-item>

Clickable row — listen for native click

<la-list-item clickable> <i slot="icon" class="ph ph-file-text"></i> Articles of Incorporation </la-list-item> const item = document.querySelector('la-list-item'); item.addEventListener('click', () => { // open detail view, navigate, etc. });

Person row — avatar + name + role + actions

<la-list-item clickable> <div slot="icon" class="avatar">AC</div> Alex Chen <span slot="meta">Founder · CEO</span> <span slot="actions"> <la-icon-button size="sm" label="Edit"> <i class="ph ph-pencil-simple"></i> </la-icon-button> </span> </la-list-item>

Error row — failed upload feedback

<la-list-item error> <i slot="icon" class="ph ph-warning"></i> Upload failed <span slot="meta">Connection lost</span> </la-list-item>

Notes

Click-stop on the actions slot — clicks inside the actions slot do not bubble to the row click. This lets you put per-row action buttons (edit, remove, more) without the row click firing whenever a user taps an action. The row click still fires for everything else — the content column, the icon, the meta.
Interactive pattern matches la-document-item — when clickable, the row is a native <button> with a ::before pseudo-element hover fill (--la-color-bg-muted) and a :focus-visible ring. See decisions/16 (native-button rows) and decisions/17 (GPU-safe hover pseudo).
Error background wins over hover fill — when error is set, the danger-surface background stays visible even on hover. The pseudo-element hover fill is suppressed so the error signal is never obscured.
Stacked lists just work — place multiple la-list-items back-to-back and the bottom separator appears between siblings; :last-child auto-removes the final border. No separator primitive needed.
Inset focus ring — the focus ring uses a negative outline-offset to sit inside the row. Rows sit flush against siblings via border-bottom, so an outset outline would overlap adjacent rows. This diverges from la-document-item's outset ring — intentional, driven by the container context.
Composition note — la-list-item covers director-row, person-row, and coverage-row patterns that appear across the founder-flow prototypes. Prefer it over handwritten <div class="row"> chrome.
