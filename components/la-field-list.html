la-field-list

A list of labelled key–value rows for document metadata, extracted facts, and entity details. Pass an array of { label, value } objects via the fields property. Each row has a bottom border to aid scanning on longer lists; the last row's border is suppressed. Two layout modes: spread (default) for narrow panels, stack for wider surfaces.

Anatomy

A <dl> wrapper with one row per field, each containing a label (<dt>) and a value (<dd>).

  1. Row — border-bottom: 1px --la-color-border-light. Last row has no border.
  2. Label (<dt>) — base size, muted colour.
  3. Value (<dd>) — base size, normal weight, primary text colour.

la-field-list

A list of labelled key–value rows for document metadata, extracted facts, and entity details. Two layout modes: spread (label left, value right-aligned) for narrow panels, and stack (fixed-width label column) for wider surfaces.

Properties

PropertyAttributeTypeDefaultDescription
fields fields FieldItem[] [] Rows to render. Each item has a `label` and a `value`.
layout layout 'spread' | 'stack' 'spread' Layout mode. `'spread'` (default) — label/value space-between. `'stack'` — fixed-width label column.

Usage

<!-- Spread (default) — document metadata -->
<la-field-list id="details"></la-field-list>
<script>
  document.getElementById('details').fields = [
    { label: 'Uploaded',   value: '3 Mar 2024' },
    { label: 'File size',  value: '88 KB' },
    { label: 'Category',   value: 'Employment' },
    { label: 'Legal area', value: 'Employment' },
  ];
</script>

<!-- Stack — company manual / wider surfaces -->
<la-field-list id="basics" layout="stack"></la-field-list>
<script>
  document.getElementById('basics').fields = [
    { label: 'Company name', value: 'Acme Corp.' },
    { label: 'Entity type',  value: 'Delaware C-Corp' },
  ];
</script>

Notes

  • Pass fields via the .fields JS property — not a string attribute. Each item is { label: string; value: string }.
  • An empty fields array renders nothing — the component is invisible. Filter out absent fields before passing.
  • Use spread (default) in narrow panels where a right-aligned value creates clear left–right scanning. Use stack on wider surfaces where a fixed-width label column aids alignment across many rows.
  • The component renders a semantic <dl> / <dt> / <dd> structure — correct for key–value data.
