la-insight-card

Surfaces a single legal risk or action item with urgency signals so users can triage at a glance. Rendered by la-legal-area-page inside the Urgent and Proactive tabs. The entire card body is clickable — wire la-insight-click to open la-insight-modal with the full detail. When other insights share the same source document, related-insight one-liners appear at the bottom of the card — clicking one fires la-insight-navigate with the target insight id instead of opening this card's modal.

la-insight-card

Surfaces a single legal risk or action item with urgency signals so users can triage at a glance. Rendered by la-legal-area-page inside the Urgent and Proactive tabs. Fires la-insight-click when the card body is clicked and la-insight-navigate when a related-insight one-liner is clicked.

Properties

PropertyAttributeTypeDefaultDescription
severity severity InsightSeverity 'urgent' Severity level — always backend-assigned. `'urgent'` or `'proactive'` for active insights; `'resolved'` for the resolved list.
headline headline string '' Plain-English headline — what the user needs to do.
description description string '' Optional one-liner shown below the headline on the card.
insightType insight-type InsightType 'action_required' Insight type — determines the resolution widget in `la-insight-modal`. Not rendered on the card.
sourceType source-type InsightSourceType 'legal' Source type — renders the type tag. `'legal'` → "Required by Law"; `'best_practice'` → "Best practice".
entity entity {name: string; type: string; id: string} | undefined — Optional entity context (person or company). Renders as a badge with icon. JS property only.
dates dates {date: string; label: string}[] | undefined — All dates associated with the insight. Card shows only `dates[0]` (nearest). Pass sorted nearest-first. JS property only.
relatedInsights relatedInsights {id: string; headline: string; legalArea: string}[] | undefined — Related insights on the same source document. Each renders as a clickable one-liner below the tags row. JS property only.
explanation explanation string '' Why this is an issue — passed through in `la-insight-click` detail for the modal. Not rendered on the card.
reason reason string '' Legacy — use `explanation`. Kept for backward compat during Sprint 5 transition. Included in `la-insight-click` detail; not rendered on the card.
action action {label: string; event: string} | undefined — Legacy — pre-Sprint-5 action API. Included in `la-insight-click` detail but not rendered.
subItems subItems GroupSubItem[] | undefined — Sub-items for grouped insights (same obligation, multiple entities). When set with 2+ items, renders an entity-count badge instead of the single `entity` badge.

Usage examples

Basic card with entity and deadline

const card = document.querySelector('la-insight-card'); card.setAttribute('severity', 'urgent'); card.setAttribute('source-type', 'legal'); card.setAttribute('headline', 'Unsigned employment contract — Jane Lee'); card.setAttribute('explanation', 'An unsigned contract...'); card.entity = { name: 'Jane Lee', type: 'person', id: 'p1' }; card.dates = [{ date: '30 Jun 2026', label: 'Contract deadline' }];

Listening for la-insight-click

card.addEventListener('la-insight-click', (e) => { // e.detail contains the full InsightData openModal(e.detail); }); // or listen on an ancestor — event bubbles page.addEventListener('la-insight-click', openModal);

Related insights — cross-insight navigation

card.relatedInsights = [ { id: 'ins_002', headline: 'Missing IP assignment for Jane Lee', legalArea: 'IP Protection', }, ]; card.addEventListener('la-insight-navigate', (e) => { // e.detail.id is the target insight to open navigateToInsight(e.detail.id); });

Setting all Sprint 5 data contract fields

card.setAttribute('insight-type', 'document_issue'); card.setAttribute('source-type', 'legal'); card.entity = { name: 'Jane Lee', type: 'person', id: 'p1' }; card.dates = [ { date: '30 Jun 2026', label: 'Contract deadline' }, { date: '1 Jul 2026', label: 'Review period ends' }, ]; card.relatedInsights = [ { id: 'ins_003', headline: '...', legalArea: 'IP Protection' }, ]; // Card shows only the nearest date (dates[0]) // Modal receives the full dates array

Notes

Severity is always backend-assigned. The card never computes its own severity. Pass severity from the insight payload — 'urgent' or 'proactive' for active insights, 'resolved' for the resolved list.
explanation is the current prop. reason is kept on the type for backward compatibility during the Sprint 5 transition. Use explanation going forward — the card does not render either prop visually, but both are included in la-insight-click event detail for the modal.
entity, dates, and relatedInsights are JS-only props. Arrays and objects cannot be set via HTML attribute. Always assign as JavaScript properties: card.dates = [...].
Only the nearest date renders on the card. The card shows dates[0] only — pass dates sorted nearest-first (backend responsibility). The full dates array is passed through in la-insight-click detail so the modal can show all dates.
Related insight clicks do not bubble to la-insight-click. The la-insight-navigate event fires with stopPropagation — only one event fires per user interaction. Wire la-insight-navigate to navigate to that insight's modal (e.g. via /insights/{id} or by swapping modal content directly if the insight is on the same page).
Source type tag copy: 'legal' → "Legal"; 'best_practice' → "Best practice". Never exposed to users as code strings — the card renders the human-readable label.
Entity icon convention: entity.type === 'person' → ph-user; any other type → ph-building. The card renders the icon inside the entity badge automatically.
