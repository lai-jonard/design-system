la-breadcrumb

A depth signal in the top bar. Shows the founder exactly where they are within the hierarchy — e.g. Dashboard / Employment / Insight detail — and provides a one-click path back up the tree. A passive secondary cue that complements the sidebar; it does not replace it.

Anatomy

Rendered inside the page header. The trail reads left to right — ancestors first, current page last.

  1. Nav wrapper — <nav aria-label="Breadcrumb">. Landmark role so screen readers announce the region.
  2. Ancestor link — <a> when href is set; <button> when omitted (fires la-breadcrumb event instead). Muted text, transitions to primary on hover.
  3. Separator — / character in faint text. aria-hidden="true" — screen readers skip it.
  4. Current page — <span aria-current="page">. Bold, primary text colour, truncates with ellipsis when space is tight. Never a link.

Usage examples

Href links — browser handles navigation

<la-breadcrumb .items=${[ { label: 'Dashboard', href: '/dashboard' }, { label: 'Employment', href: '/employment' }, { label: 'Insight detail' } ]} ></la-breadcrumb>

Programmatic — handle navigation in JS

<la-breadcrumb .items=${[ { label: 'Dashboard' }, { label: 'Employment' }, { label: 'Insight detail' } ]} @la-breadcrumb=${(e) => router.go(e.detail)} ></la-breadcrumb>

Via la-app-shell — pass through the shell's breadcrumbs prop

<la-app-shell show-header .breadcrumbs=${[ { label: 'Dashboard', href: '/dashboard' }, { label: 'Governance', href: '/governance' }, { label: 'Documents' } ]} > ... </la-app-shell>

Listening for the event on la-app-shell

shell.addEventListener('la-breadcrumb', e => { const { label, index } = e.detail; router.navigateTo(index); });

Notes

The three wayfinding surfaces must agree — the sidebar active state, the breadcrumb, and the page title should all reflect the same location. A user should never be in a state where the sidebar shows one thing and the breadcrumb implies another.
Breadcrumb is a passive cue, not the primary nav — it complements the sidebar's role. The sidebar is always visible and provides the primary spatial anchor. The breadcrumb provides depth context and a shortcut back up the tree for users who arrived via a deep link or multi-step flow.
Last item is never a link — the current page item is always a <span aria-current="page">. Linking the current page serves no purpose and creates a confusing interaction. Enforce this by ensuring the last item in the items array has no href.
Use href or programmatic — not both — pick one navigation strategy for the whole app. Mixing native href links with event-based navigation in the same trail will produce inconsistent browser behaviour (back-button state, history entries).
Current page label truncates at the right edge — text-overflow: ellipsis applies to the current page span only. Ancestor labels do not truncate — they push the current page label toward the edge. Keep ancestor labels short.
