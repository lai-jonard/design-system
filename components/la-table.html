la-table

A horizontal-scroll wrapper around a consumer-provided <table>. HTML parsing strips table-structure tokens before they reach shadow DOM, so the table lives in the light DOM, slotted inside la-table. Load la-table.css in the page head — it targets la-table table selectors in the light DOM and applies all default styles.

Anatomy

Full-width table: muted header row, comfortable body cells, bottom border suppressed on the last row.

la-table

A horizontal-scroll wrapper around a consumer-provided <table>. HTML parsing strips stray <thead>/<tbody>/<tr>/<th>/<td> tokens before they reach shadow DOM, so la-table cannot host table structure directly — consumers nest a real <table> inside. Load la-tokens-root.css and la-table.css once in the page and the nested table inherits the full visual rhythm: muted header row, comfortable body padding, hover highlight, and success-row tint.

Usage examples

Minimal — default visual rhythm

<la-table> <table> <thead><tr> <th>Document</th> <th>Status</th> </tr></thead> <tbody> <tr> <td>Articles of Incorporation</td> <td><la-badge variant="success">Verified</la-badge></td> </tr> </tbody> </table> </la-table>

Success row — mark verified/resolved rows

<tr class="success"> <td>IP Assignment — Jamie Lee</td> <td><la-badge variant="success">Verified</la-badge></td> </tr> <!-- or attribute form: --> <tr data-success>…</tr>

Required stylesheet imports

<link rel="stylesheet" href="/la-tokens-root.css"> <link rel="stylesheet" href="/la-table.css">

Notes

Why not shadow DOM? — HTML parsers treat <thead>, <tbody>, <tr>, <th>, and <td> as invalid outside of a <table> and silently hoist or drop them before they reach the custom element. The wrapper pattern is the only reliable way to accept table children in a web component.
Success row — class vs attribute — both .success and [data-success] apply the tint. The class form is convenient for JS toggling; the attribute form reads cleanly in static markup or server-rendered templates. Either works; pick one consistently per table.
Hover is CSS-only — tbody tr:hover td tints the row with --la-color-bg-subtle. No JS needed. Success rows honour the hover rule too — they tint over the success background on hover.
Horizontal scroll — la-table sets overflow-x: auto on its internal wrapper. Set min-width on the inner <table> when columns should never collapse below a readable width.
Empty state — la-table has no built-in empty state. Render a single <tr><td colspan="N"> with your empty-state copy, or conditionally swap the entire component for la-empty-state.
