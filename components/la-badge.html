la-badge

A small inline label for status, category, severity, or count. Two types: badge (pill, for status labels) and counter (circle, for numerical counts). Always paired with content — never standalone.

Anatomy

Badge is a pill with slotted label content. Counter is a circle with a number.

Expiring soon 3
  1. Pill container — badge type. Shared --la-color-bg-subtle surface with a 1px --la-color-border-light border and --la-radius-pill. Variant sets the text colour; primary overrides both bg and border to the solid primary colour.
  2. Label — default slot. A word or short phrase.
  3. Counter circle — counter type. Fixed 20px height, --la-radius-circle, no border. Variant sets the fill; appearance switches between solid (filled) and tinted-surface (ghost).

Usage examples

Status label — urgent

<la-badge variant="danger"> Action required </la-badge>

Category label

<la-badge variant="accent"> Doc Quality </la-badge>

With icon — fill weight only

<la-badge variant="warning"> <i slot="icon" class="ph-fill ph-warning-circle"></i> Expiring </la-badge>

Counter — filled (unread count)

<la-badge type="counter" variant="primary"> 12 </la-badge>

Counter — ghost (subtle indicator)

<la-badge type="counter" variant="neutral" appearance="ghost"> 3 </la-badge>

Counter — danger (urgent count)

<la-badge type="counter" variant="danger"> 5 </la-badge>

Notes

Badge is always paired with content, never standalone — a badge outside of a containing row, card, or title loses its meaning. The surrounding content is what it's labelling.
Counter is a numerical count signal — use it for unread item counts, queue depths, and notification totals. For single-digit counts it renders as a perfect circle; for two or more digits the width grows to accommodate the number.
Variant names carry accessible context — colour alone fails a11y. The semantic variant name (danger, warning, etc.) pairs with the colour so meaning is not colour-only. Logged in design-language/components.md known divergences.
accent is a category, not a status — use it for classification labels like "Doc Quality", "Legal Compliance". The other variants carry status or prominence meaning; accent carries domain meaning. Not applicable to counter type.
Ghost appearance is for deemphasised counters — ghost uses the faint tinted surface for the variant colour rather than a solid fill, giving a softer presence. Use it when the count should not dominate the surrounding UI. Use filled (default) when the count is a primary signal such as an unread inbox badge.
Icons use Phosphor fill weight — when slotting an icon on a badge, always use ph-fill, never the outline ph class. The filled glyph matches the badge's solid tinted surface.
