la-avatar

Initials avatar for people and entities. Derives two-letter initials automatically from a full name, with an optional override. Three fixed sizes — sm (32 px), md (40 px), lg (48 px) — map to specific product contexts. Purely presentational: no interactions, no events.

Anatomy

A single circular surface holding up to two uppercase initials. Background is --la-color-bg-muted with a 1 px --la-color-border-light border so it reads on both white and off-white surfaces.

la-avatar

Initials avatar for people and entities. Auto-derives two-letter initials from a full name, with an override. Three fixed sizes (sm/md/lg). Purely presentational.

Properties

PropertyAttributeTypeDefaultDescription
name name string '' Full name used to auto-derive initials (e.g. "James Okafor")
initials initials string '' Override for auto-derived initials (e.g. "JO")
size size 'sm' | 'md' | 'lg' 'md' 'sm' | 'md' | 'lg'. Default: 'md'
_displayInitials — string — —

CSS custom properties

No public CSS custom properties. Size and colour are fixed to design-token values for consistency. If you need a non-standard size, use a wrapping element with transform: scale() rather than forking the component.

Usage examples

Auto-derived initials

<la-avatar name="James Okafor"></la-avatar>
<!-- renders "JO" -->

Initials override

<la-avatar initials="AC" size="lg"></la-avatar>

In a detail panel header

<div class="inv-header">
  <la-avatar name="${investor.name}" size="lg"></la-avatar>
  <div class="inv-identity">
    <div class="inv-name">${investor.name}</div>
    <div class="inv-firm">${investor.firm}</div>
  </div>
</div>

Notes

  • Initials derivation: splits on spaces, takes the first character of the first two words. Single-word names produce a single-letter initial. Names with prefixes (e.g. "de la") may produce unexpected results — use the initials override.
  • Accessibility: the circle is marked aria-hidden="true" — it's decorative alongside a visible name. If used standalone (e.g. in a list with no adjacent name text), add an aria-label to the wrapper element.
  • Colour: background and border are fixed to token values. Do not override — the neutral palette ensures avatars don't compete with status colours (badges, dots). Future work: optional colour-seeded backgrounds per identity (decision pending).
  • Do not use for company logos or uploaded photos — this component is initials-only. Photo/logo support is a separate component decision.
