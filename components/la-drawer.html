Usage examples

Basic — right edge with heading

<la-drawer open heading="Documents"> <p>Everything we've reviewed so far.</p> </la-drawer>

Detail inspector — heading + description

<la-drawer edge="right" heading="Employee details" description="Roles, documents, and history" ?open=${isInspectorOpen} > <!-- inspector content --> </la-drawer>

Mobile navigation — left edge

<la-drawer edge="left" heading="Menu"> <la-sidebar></la-sidebar> </la-drawer>

Backdrop dismiss — click outside to close

<la-drawer backdrop-dismiss heading="Quick filter" > <!-- filter chips --> </la-drawer>

Custom header — replaces heading, keeps close button

<la-drawer> <div slot="header"> <i class="ph ph-folder"></i> <span>Documents</span> </div> <!-- body --> </la-drawer>

With footer actions

<la-drawer heading="Edit details"> <!-- form fields --> <div slot="footer"> <la-button variant="secondary">Cancel</la-button> <la-button variant="primary">Save</la-button> </div> </la-drawer>

Listening for close

const drawer = document.querySelector('la-drawer'); drawer.addEventListener('close', () => { // drawer has already flipped open = false // sync any related UI state here });

Custom size via token override

la-drawer.wide { --la-drawer-size: 480px; }

Notes

Always-on dismissal — Escape and the close button always close the drawer. Backdrop click is opt-in via the backdrop-dismiss attribute; this protects users from losing progress during multi-step tasks.
Scroll lock — the page behind the drawer is scroll-locked while the drawer is open by default. Set no-scroll-lock to keep the page scrollable (for reference-style drawers where users cross-check content).
Focus management — on open, focus moves to the first focusable element inside the panel (the close button by default). Tab and Shift+Tab cycle within the panel. On close, focus returns to the element that opened the drawer.
Mobile full-width — right and left drawers become 100vw below a 480px viewport. Top and bottom drawers are already full-width at every viewport. Corner radius collapses to zero in full-width mode.
Header pattern — the close button is always rendered in the top-right regardless of edge. If the header slot is populated, it replaces the prop-driven heading block; the close button stays. If the slot is empty and no heading is set, only the close button is shown.
Composition in la-app-shell — la-app-shell composes la-drawer internally for its aside panel on narrow viewports. If you're rendering an aside through la-app-shell, you don't need to instantiate la-drawer yourself — just slot content into slot="aside" and rely on la-toggle-aside events.
