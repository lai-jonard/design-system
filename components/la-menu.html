la-menu

Tertiary action surface — a 3-dot trigger button that reveals a floating list of labelled actions. Reach for la-menu when an action doesn't warrant a visible button and isn't the primary CTA: rename, delete, export, share, and similar context-specific commands. la-menu-item is the row primitive slotted inside it.

Anatomy

la-menu — trigger + panel
la-menu regions
RegionSourceRole
Trigger la-icon-button 32×32 ghost icon button. Always ph-bold ph-dots-three. Carries aria-label, aria-haspopup="menu", and aria-expanded. Hover + open-state: --la-color-bg-muted background.
Panel .panel role="menu" Floating surface positioned relative to the trigger. Scales in from the corner set by origin. Holds the default slot for la-menu-item children.
Slot <slot> Default slot inside the panel. Accepts la-menu-item, la-menu-toggle, and la-menu-divider children.
la-menu-item
RegionSourceRole
Row button.item role="menuitem" Full-width button. 32px tall. Transparent background, --la-color-bg-subtle on hover. --la-radius-sm.
Icon span.icon (optional) Phosphor icon at --la-font-size-base. Only rendered when the icon property is set. Inherits danger colour on variant="danger".
Label span.label Display text. Always sentence case. Never truncated — keep labels short.

The panel duplicates the .t-dropdown motion pattern from motion.css inside the shadow root — page-scope CSS does not penetrate shadow DOM, so the rules are mirrored using the same motion tokens. Reduced-motion users see no panel transition.

Usage examples

Basic — listen on the menu for selection

<la-menu label="Document options"> <la-menu-item label="Edit" icon="ph ph-pencil-simple" value="edit"></la-menu-item> <la-menu-item label="Duplicate" icon="ph ph-copy" value="duplicate"></la-menu-item> <la-menu-item label="Delete" icon="ph ph-trash" value="delete" variant="danger"></la-menu-item> </la-menu> <script> document.querySelector('la-menu').addEventListener('la-menu-select', e => { console.log(e.detail.value); // 'edit' | 'duplicate' | 'delete' }); </script>

Left-aligned trigger (trigger on the left side of a surface)

<la-menu origin="top-left" label="Row options"> <la-menu-item label="Archive" value="archive"></la-menu-item> <la-menu-item label="Remove" value="remove" variant="danger"></la-menu-item> </la-menu>

With divider and toggle

<la-menu label="More options"> <la-menu-item label="Export" icon="ph ph-export" value="export"></la-menu-item> <la-menu-divider></la-menu-divider> <la-menu-toggle label="Show resolved items" value="show-resolved" ></la-menu-toggle> </la-menu> <script> document.querySelector('la-menu').addEventListener('la-menu-toggle-change', e => { console.log(e.detail.value, e.detail.checked); }); </script>

Partner: la-menu-item

Row primitive slotted inside la-menu. Renders a full-width <button role="menuitem">. Fires la-menu-select on click and automatically closes the parent panel.

Properties

PropertyAttributeTypeDefaultDescription
label label string '' Display text for the row. Also used as detail.label in the la-menu-select event. Sentence case always.
value value string '' Identifier passed in la-menu-select detail. Falls back to label if not set — provide an explicit value when the label text is liable to change.
icon icon string '' Phosphor icon class string, e.g. 'ph ph-pencil-simple'. Optional — omit for text-only rows. If some items in a menu have icons, all should.
variant variant 'default' | 'danger' 'default' Use danger for destructive actions. Renders label and icon in --la-color-danger-text with a --la-color-surface-danger hover. Always place the danger item last.
disabled disabled boolean false Renders the item at 40% opacity and removes pointer events. Use for temporarily unavailable actions — prefer hiding the item entirely when it can never be available in the current context.

Events

EventDetailWhen fired
la-menu-select {label: string, value: string} On click. Bubbles and composed — listen on la-menu or any ancestor. Fires before la-menu-close.

la-menu-toggle — toggle row

A switch row for boolean settings. Uses la-switch internally. Does not close the panel on change — designed for in-menu settings that the user toggles multiple times. Fires la-menu-toggle-change with {label, value, checked}.

la-menu-divider — horizontal rule

Visual separator between menu sections. Renders an <hr role="separator"> with --la-color-border-light stroke and --la-space-xs vertical padding. No properties.

Notes

  • Danger last. Always put the destructive item at the bottom of the list. It should never be the first thing a user sees.
  • Keep it short. 3–6 items is the comfortable range. More than that suggests the surface needs a different pattern.
  • Icon consistency. If any item in a menu has an icon, all items should. Mixed icon/no-icon lists read unbalanced.
  • Sentence case. "Export as PDF", not "EXPORT AS PDF" or "Export As PDF".
  • Keyboard. Escape closes the panel and returns focus to the trigger. Click-outside closes without moving focus. Tab proceeds naturally through items — no arrow-key roving focus is implemented (appropriate for short lists).
  • Origin default is top-right. Right for the majority of card and row 3-dot triggers. Switch to top-left when the trigger is on the left side of a surface, or bottom-* when near the viewport bottom.
  • Disabled vs hidden. Prefer hiding items that can never be available in the current context. Disabled is for temporarily unavailable actions — always pair with context elsewhere so the user understands why.
