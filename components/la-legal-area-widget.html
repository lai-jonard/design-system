la-legal-area-widget

Dashboard tile for a single legal area. Three states — unlocked, locked-unlockable, locked-not-unlockable — render different chrome inside a fluid 4:3 landscape card (300 px wide minimum, height tracks via aspect-ratio). The unlocked state surfaces insight counts or a next-level CTA; the unlockable state surfaces the next action; the not-unlockable state names the dependency holding it back.

Anatomy

One la-panel filling the host. The illustration region takes the slack at the top (flex: 1); title, description, counts/CTA stack at the bottom of the panel padding so the layout reads from the artwork down into the actionable content.

Usage examples

Dashboard 3-up grid — fluid columns

<div style="display:grid;grid-template-columns:repeat(3,1fr);gap:24px;"> <la-legal-area-widget area="Governance" description="Founding documents, directors, registered agent." status="unlocked" urgent="2" proactive="3" next-level-label="Continue with Governance" illustration="/assets/illustrations/empty-state.png" ></la-legal-area-widget> <!-- Employment, Fundraising follow in fixed order --> </div>

Wire navigation — both events route to the area page

grid.addEventListener('la-area-click', (e) => { router.go(`/legal-areas/${e.detail.area.toLowerCase()}`); }); // la-area-unlock fires from both the locked-unlockable // "Start with [area]" CTA and the unlocked next-level CTA. // Both mean "user wants the next progression for this area". grid.addEventListener('la-area-unlock', (e) => { router.go(`/legal-areas/${e.detail.area.toLowerCase()}`); });

Locked-not-unlockable — name the dependency

<la-legal-area-widget area="Fundraising" description="Cap table, DD readiness, investor data." status="locked-not-unlockable" dependencies='["Governance"]' ></la-legal-area-widget>

Notes

Fluid 4:3 landscape, 300 px width floor. Width comes from the parent grid; height tracks via aspect-ratio: 4 / 3. The 300 px min-width stops the card collapsing below readable size; the dashboard grid is the responsive surface.
Surface mapping. unlocked + locked-unlockable sit on white (--la-color-bg) so the illustration reads cleanly. locked-not-unlockable keeps the default la-panel grey — the multiply-blended illustration sinks into it.
locked-unlockable rests with shadow-card emphasis. The card a user can act on right now gets --la-shadow-card at rest, hover lifts it to --la-shadow-md. unlocked rests flat and lifts to --la-shadow-sm. Same emphasis pattern as the "next progression candidate" rule.
Illustration treatment. Full opacity on white surfaces (unlocked + locked-unlockable); filter: grayscale(1) + mix-blend-mode: multiply on the locked-not-unlockable grey surface so the white halo dissolves into the card.
Empty illustration collapses gracefully. When no URL is passed, the illustration region disappears entirely and content rises to the top of the card. Documented in the "Without illustration" snapshot.
"All good" instead of "0 urgent · 0 proactive". When an unlocked area has zero counts and no nextLevelLabel, the card shows a green check and "All good" — positive context rather than visual silence. When a next-level CTA is set, the CTA wins the slot (decision 54).
Next-level CTA on unlocked cards. Setting nextLevelLabel on an unlocked card renders a primary la-button[size=sm] ("Continue with…") and suppresses the "All good" line. CTA fires la-area-unlock like the locked-unlockable button.
No level codes in user-facing copy. Pass full area names ("Continue with Governance", "Requires Governance") — never L0/L1/L2 (decision 56). Internal taxonomy stays in code; user copy stays in plain area names.
locked-not-unlockable is dimmed but still fires events. The visual treatment via la-panel ?dimmed signals non-interactivity, but la-area-click still fires. Consumers decide what to do with the click — typical use is to ignore or route to a tooltip explaining the dependency.
Single primary CTA per dashboard at a time. Visual emphasis comes from --la-shadow-card on the locked-unlockable card and the primary CTA inside it — not a separate "Recommended" badge. Earlier sprints proxied this via a recommended prop; that prop is gone, the surface emphasis is enough.
Keyboard activation. The la-panel surface carries tabindex="0" + role="button" + an aria-label ("Open area"). Enter and Space fire la-area-click. Focus shows a 2 px outline using --la-color-border-focus.
No icon in the title row. Areas are named in text only — no per-area icon. Keeps the dashboard grid visually consistent.
