Chat

Two primitives that assemble into a conversational AI interface. la-chat-message renders individual messages — user bubbles (right, dark) and assistant responses (left, plain). la-chat-input is the composer bar with an auto-growing textarea, send button, and loading state. Neither component owns state — the host page drives the conversation.

Assembled demo

Type a message and press Enter or Send. A simulated assistant response appears after a short delay.

la-chat-message

A single conversational turn. Adapts layout and visual style based on the sender attribute — "assistant" messages are left-aligned with a small indicator circle; "user" messages are right-aligned with a dark inverted bubble.

Properties

PropertyAttributeTypeDefaultDescription

la-chat-attachment

Compact document reference card for use inside la-chat-message via slot="attachments". Reuses the ready | processing | failed | none status vocabulary from la-document-item. When href is set, the card renders as an <a> that opens the document in a new tab.

Anatomy
RegionRole
Icon container 32 × 32 px rounded square with --la-color-bg-subtle. Holds ph-file-text at xl size, or a spinner when status="processing".
Status badge 13 px filled icon overlaid bottom-right of the icon container. Green check-circle for ready, red x-circle for failed. Not shown for none or processing.
Document name Medium weight, truncated with ellipsis. Full name available via the title attribute tooltip.
Sub-label category and association joined with " · ". Shown only when at least one is set.
Chevron Right-arrow icon. Shown only when href is set, signalling the card is navigable.

Properties — la-chat-attachment

PropertyAttributeTypeDefaultDescription
name name string '' Document title. Truncated with ellipsis if it overflows; full title shown via native title tooltip.
category category string '' Optional category label — e.g. "Fundraising", "Governance". Shown in the sub-label line.
association association string '' Optional entity name the document belongs to. Mirrors the association field on la-document-item. Shown in the sub-label line.
status status 'none' | 'ready' | 'processing' | 'failed' 'none' Processing state. none — no badge. ready — green check-circle. processing — spinner replaces file icon. failed — red x-circle. Matches la-document-item vocabulary.
href href string '' When set, renders the card as an <a> with this href (opens in a new tab). A right-chevron is shown. When empty, the card is a non-interactive <div>.

la-chat-input

Composer bar for the chat interface. Auto-growing textarea with a primary send button. Handles Enter-to-send (Shift+Enter inserts a newline). Resets after dispatch. Provides a loading state that shows a spinner and disables input while a response is in-flight.

la-chat

Composed shell that manages the thread + input bar in a single element. Pass a messages array and handle la-send — the component renders each message, auto-scrolls to the bottom on update, and exposes slots for empty state, notice banners, and read-only replacements.

Live demo

Type a message and press Enter. Replies simulate a 1.4 s server round-trip.

State variants

With messages — founder ↔ investor thread
With attachment — document shared in thread
Sending state — message in flight
Read-only — access revoked (default notice)
Empty state — no messages yet

Usage examples

Minimal assembled chat

<div class="chat-shell">
  <div class="thread" id="thread">
    <la-chat-message sender="assistant">
      How can I help you today?
    </la-chat-message>
  </div>
  <la-chat-input id="input" placeholder="Ask a question…"></la-chat-input>
</div>

<script>
  var input = document.getElementById('input');
  var thread = document.getElementById('thread');

  input.addEventListener('la-submit', async function(e) {
    // 1 — append user message
    var userMsg = document.createElement('la-chat-message');
    userMsg.sender = 'user';
    userMsg.textContent = e.detail.value;
    thread.appendChild(userMsg);

    // 2 — show typing indicator
    input.loading = true;
    var typing = document.createElement('la-chat-message');
    typing.sender = 'assistant';
    typing.typing = true;
    thread.appendChild(typing);

    // 3 — fetch response, replace typing with result
    var response = await fetchAIResponse(e.detail.value);
    typing.remove();
    var reply = document.createElement('la-chat-message');
    reply.sender = 'assistant';
    reply.textContent = response;
    thread.appendChild(reply);
    input.loading = false;
  });
</script>

Streaming response

// Replace the typing message with a streaming one once the first token arrives
typing.remove();
var reply = document.createElement('la-chat-message');
reply.sender = 'assistant';
reply.streaming = true;
thread.appendChild(reply);

for await (var chunk of stream) {
  reply.textContent += chunk;
}
reply.streaming = false;
input.loading = false;

In a Lit template

html\`
  <la-chat-message
    sender=${msg.sender}
    ?streaming=${msg.isStreaming}
    ?typing=${msg.isTyping}
    .timestamp=${msg.time}
  >${msg.text}</la-chat-message>
\`

Notes

  • Stateless by design: Neither component manages a message list. The host page owns the conversation array and renders messages. This keeps the components composable and testable in isolation.
  • Auto-grow textarea: la-chat-input grows to a max of 180 px before scrolling. The component resets height after submit — no post-submit cleanup needed in the host page.
  • typing vs streaming: Use typing while waiting for the first token (no content yet). Switch to streaming once tokens arrive and content is being appended. Both are screened by prefers-reduced-motion.
  • ARIA: The typing indicator has role="status" so screen readers announce when the assistant starts responding. The streaming cursor is aria-hidden. Add aria-label to the thread container for overall context.
  • Sender attribute: Named sender rather than role to avoid collision with the HTML role ARIA attribute.
  • Attachments and document status: la-chat-attachment shares the same ready | processing | failed | none status vocabulary as la-document-item so status can be passed through directly from DD pack data. Multiple attachments stack in order. If an attachment needs a click handler rather than a static href, wrap it in a <button> or listen to click in the host page.
  • Human conversations: Set name on left-side messages to switch from the AI sparkle to initials. The same sender="user" / sender="assistant" layout logic applies — "user" is always the current viewer, "assistant" is the other party (AI or human).
