la-upload

Dual-source upload zone for hosting contexts. Renders Browse files + Google Drive buttons above a drop zone, with a separate transfer-failure alert that hosts drive on network errors. Composes la-upload-box for the drop-zone surface itself — drag-and-drop, click-to-browse, type / size / multi-file validation, and file-rejection error state all live there.

Three regions stacked vertically. Sources row + optional transfer-error alert + drop zone.

Dual-source upload zone for hosting contexts. Renders Browse files + Google Drive buttons above a drop zone, with a separate transfer-failure alert that hosts drive on network errors. Usage examples

Basic — listen for a valid file

<la-upload accept=".pdf,.docx" max-size="52428800" hint="PDF or DOCX, up to 50 MB." ></la-upload> const upload = document.querySelector('la-upload'); upload.addEventListener('la-change', e => { startUpload(e.detail.file); });

Drive OAuth — host owns the popup

upload.addEventListener('la-upload-drive', () => { openDriveOAuthPopup().then(token => { openPicker(token).then(driveFile => { startUpload(driveFile); }); }); });

Transfer-failure flow

upload.processing = true; fetch(...).catch(() => { upload.processing = false; upload.transferError = true; upload.transferErrorMessage = 'Upload interrupted. Check your connection and try again.'; }); upload.addEventListener('la-retry-transfer', () => { upload.transferError = false; retry(); });

Handle file rejection

upload.addEventListener('la-reject', e => { if (e.detail.reason === 'multi') { /* la-upload-box already shows the inline message */ } if (e.detail.reason === 'size') { /* analytics, telemetry, etc. */ } });

Notes

Composition over inheritance. la-upload wraps la-upload-box. The drop zone, validation, file-rejection error state, and upload spinner all live in the inner box; this layer adds the Drive button and the transfer-failure alert. Reach for la-upload-box directly when you need the drop zone alone (no Drive button, no host-driven transfer-error). See decisions/48-upload-compose-not-extend.md.
Drive OAuth is a host concern. The Drive button only fires la-upload-drive. The host owns the popup, picker, token caching, and silent re-consent on token expiry. Sprint 1 keeps Drive tokens local in the existing LAI.GoogleApi module — the component stays unaware.
Two error categories, never simultaneous. File-rejection (type / size / multi-file) is owned by la-upload-box and stays inside its own surface with a re-select affordance. Transfer-failure (network / timeout) is host-driven via transfer-error with its own Retry affordance above the zone. The two states model different failure modes and surface differently. See decisions/49-upload-two-tier-error-model.md.
Multi-file drop is rejected. Sprint 1 is single-file v1. Dropping more than one file fires la-reject with detail.reason: 'multi' and shows "Only one file can be uploaded at a time." in the drop zone.
Both source buttons are equal weight. Browse files and Google Drive both render as variant="secondary" — neither is biased as the primary action because they are parallel choices. The actual primary in any host context is whatever submit / resolve action the host modal owns.
Typical hosts. Insight-modal evidence upload, add-person avatar, cap-table import, unlock-step document attachment. Each host sets its own accept per Sprint 1 (PDF / DOCX default, PDF / DOCX / XLSX / CSV for cap table; 50 MB across the board).
compact for space-constrained hosts. Setting compact switches the drop zone to a slim horizontal layout — icon and prompt inline on a single row, reduced padding, hint hidden. Passes through to la-upload-box. Use in modals and side panels where the standard tall drop zone would dominate the content area. All behaviour (drag-drop, file selection, processing state) is identical.
