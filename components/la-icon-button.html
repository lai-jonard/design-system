la-icon-button

Icon-only trigger for secondary actions in dense surfaces — widget headers, toolbars, sidebar collapse. Shares the --la-button-height-* scale with la-button so mixed rows align. Requires a label for screen readers.

Properties

PropertyAttributeTypeDefaultDescription
label label string '' —
expanded expanded boolean | undefined — —
pressed pressed boolean | undefined — Toggle state — set when the button acts as a two-state toggle. Reflects as attribute; drives `aria-pressed` and active visual.
size size 'default' | 'sm' | 'lg' 'default' —
variant variant 'default' | 'primary' | 'outline' | 'danger' 'default' —
disabled disabled boolean false —

CSS tokens

All sizing and colour values come from the design token system defined in src/styles/tokens.ts. Override per-instance via style or by targeting the ::part(button) element.

Usage examples

Basic — dismiss action

<la-icon-button label="Dismiss"> <i class="ph ph-x"></i> </la-icon-button>

Small — widget header

<la-icon-button size="sm" label="More options"> <i class="ph-bold ph-dots-three"></i> </la-icon-button>

Pressed — on/off toggle

<la-icon-button label="Show resolved items" pressed > <i class="ph-fill ph-check-circle"></i> </la-icon-button>

Expanded — disclosure trigger

<la-icon-button label="Collapse sidebar" expanded > <i class="ph ph-sidebar-simple"></i> </la-icon-button>

Disabled

<la-icon-button label="Unavailable" disabled > <i class="ph ph-x"></i> </la-icon-button>

Click handler

const btn = document.querySelector('la-icon-button'); btn.addEventListener('click', () => { // toggle expanded state btn.expanded = !btn.expanded; });

Destructive — danger variant

<la-icon-button variant="danger" label="Delete document" > <i class="ph ph-trash"></i> </la-icon-button>

Square corners — override the default circle

/* Target the inner <button> via part */ la-icon-button::part(button) { border-radius: 0; }

Notes

Fully rounded by default — la-icon-button uses --la-radius-circle (50%) so the button renders as a circle on every size. Override via ::part(button) if you need square corners (see usage example).
Four variants — default is the most common choice (transparent with hover fill, used in widget headers and toolbars). primary and danger are for high-emphasis actions — use sparingly. outline sits between them for secondary emphasis.
Bold weight for dots-three — Phosphor's ph-dots-three renders too thin at our button sizes. Use ph-bold ph-dots-three instead. Other Phosphor icons stay at regular weight. Logged in CLAUDE.md.
Label is required — la-icon-button has no visible text. Without a label prop, screen readers announce nothing. The component warns in the console ([la-icon-button] Missing label) on first render if the prop is omitted or empty — treat this as a required field.
Sibling to la-button — use la-button for text-labelled actions. la-icon-button is specifically for icon-only triggers in dense surfaces. If you need an icon and text, use la-button with its icon-left or icon-right slot.
Shared size scale — the three sizes use --la-button-height-* tokens shared with la-button. A size="sm" text button and a size="sm" icon button in the same row will align at 28px. Logged in decisions/12.
pressed vs expanded — use pressed for binary on/off toggles (e.g. "Show resolved items", mute, bookmark). Sets aria-pressed and applies --la-color-bg-active as the background. Swap the icon between outline (ph ph-*) and fill (ph-fill ph-*) in the slot to reinforce the state visually. Use expanded for disclosure triggers (dropdown, sidebar collapse) — it sets aria-expanded and does not apply a persistent active background.
Expanded attribute quirk — to set expanded=false you need JS property assignment (el.expanded = false). HTML attribute presence is always truthy, so expanded="false" as an attribute does not work. Leave the attribute off for "not a toggle", or bind it dynamically in your framework.
