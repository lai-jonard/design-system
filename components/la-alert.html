Usage examples

Basic — info variant

<la-alert variant="info"> Your documents are being reviewed. </la-alert>

Warning — surface an expiring item

<la-alert variant="warning"> Two contracts are expiring in the next 30 days. </la-alert>

With icon slot — leading glyph

<la-alert variant="warning"> <i slot="icon" class="ph-fill ph-warning-circle"></i> Two contracts are expiring soon. </la-alert>

Neutral — context only

<la-alert variant="neutral"> This section is read-only. </la-alert>

Notes

Persistent, not dismissible — there is no close control. The alert stays visible until the state it describes changes. For transient feedback that self-dismisses, reach for la-toast instead.
Icon slot is optional but the gap still applies — the inner flex container uses gap: 10px. Leaving the icon slot empty still reserves the gap between the icon region and the body. For variants where an icon adds meaning (warning, danger), include one; for pure text alerts the empty gap is harmless.
Always use Phosphor fill weight — icons inside la-alert use ph-fill, not the outline ph class. The filled glyph matches the alert's solid tinted surface and reads as a single emphatic unit; an outline icon against the tinted background feels visually disconnected.
Stacks in narrow containers — below 240px container width, the alert switches to column layout: icon above, body below. Driven by a container query on the host, so behaviour adapts to the parent's available width without any prop or viewport check.
Colour is not the only channel — per design-language policy, status is carried by both colour and copy framing. Do not rely on colour alone to communicate meaning — the body text should read as informative even in monochrome.
